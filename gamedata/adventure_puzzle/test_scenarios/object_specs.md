# Спецификации объектов GameMaker 2 проекта

## Обзор

Этот документ описывает спецификации для объектов и скриптов, созданных для миграции "Adventure Puzzle Game" в GameMaker Studio 2.

## Объект игрока (obj_player)

### Спецификации инициализации
- `current_state` должен быть `"normal"` по умолчанию
- `speed` должен быть равен `player_config.speed_normal` (150)
- `facing_direction` должен быть 0 (вниз) по умолчанию
- `interaction_enabled` должен быть `true`
- `solid` должен быть `true` для физических взаимодействий
- `depth` должен быть `-1` для отображения поверх других объектов

### Спецификации движения
- При нажатии клавиш WASD объект должен перемещаться по экрану
- Скорость: `speed_normal` (150) для ортогонального движения, `speed_diagonal` (100) для диагонального
- При диагональном движении применяется коэффициент 0.7071 для нормализации скорости
- `facing_direction` должен обновляться в соответствии с последним направлением движения:
  - 0: вниз
  - 1: влево
  - 2: вправо
  - 3: вверх

### Спецификации взаимодействия
- Взаимодействие происходит по нажатию клавиши 'E' или Enter
- Объект должен искать ближайший интерактивный объект в радиусе 96 пикселей
- При успешном взаимодействии должен проигрываться звук `snd_ui_confirm`
- При отсутствии доступных объектов должна отображаться надпись "Здесь не с чем взаимодействовать"

### Спецификации состояний
- `"normal"`: обычное состояние, позволяет движение и взаимодействие
- `"puzzle"`: состояние головоломки, игрок невидим (`visible=false`), без коллизии (`solid=false`)
- `"menu"`: состояние меню
- `"dialog"`: состояние диалога, взаимодействие отключено

## Головоломка лабиринта (obj_maze_puzzle)

### Спецификации инициализации
- `grid` должен быть размером `width x height` (по умолчанию 15x15)
- Все ячейки `grid` изначально являются стенами (`true`)
- Начальная позиция игрока: `(start_x, start_y)` (обычно 1, 1)
- Позиция выхода: `(exit_x, exit_y)` (обычно width-2, height-2)
- `current_state` должен быть `"active"`
- `solved` должен быть `false`

### Спецификации генерации
- Лабиринт должен генерироваться по алгоритму DFS (поиск в глубину) с возвратом
- Должен существовать проходимый путь от начальной точки до выхода
- Генерация должна учитывать нечетные размеры сетки
- Выход всегда должен быть доступен (не стеной)

### Спецификации скольжения
- Игрок должен "скользить" до ближайшей стены в выбранном направлении
- Движение останавливается при достижении стены, края поля или диагональной стены
- При скольжении не должно происходить прохождения сквозь стены

### Спецификации завершения
- Когда `player_x == exit_x` и `player_y == exit_y`, головоломка считается решенной
- `solved` становится `true`
- `current_state` изменяется на `"solved"`
- Должен проигрываться звук `snd_puzzle_solve`
- Должно отображаться уведомление "Головоломка решена! Вы достигли выхода."

## Головоломка поиска слов (obj_word_search_puzzle)

### Спецификации инициализации
- `grid` должен быть размером `width x height` (по умолчанию 12x12)
- `puzzle_config.words_to_find` должен содержать список искомых слов
- `words_found` должен быть пустым массивом
- `current_state` должен быть `"phase1_search"`
- Все слова должны быть размещены в сетке до начала игры

### Спецификации размещения слов
- Слова должны размещаться по горизонтали, вертикали или диагонали
- Слова могут пересекаться
- Неиспользованные ячейки должны заполняться случайными буквами
- Все искомые слова должны быть представлены в сетке

### Спецификации выделения
- Игрок должен иметь возможность выделять линию букв кликом на первую и последнюю букву слова
- Выделение должно быть валидным только по прямой линии (горизонталь, вертикаль, диагональ)
- Найденное слово должно проверяться на соответствие одному из искомых слов

### Спецификации завершения
- Головоломка считается завершенной, когда найдены все искомые слова
- `solved` становится `true`
- `current_state` изменяется на `"solved"` или `"phase2_memory"`
- Проигрывается звук `snd_puzzle_solve`

## Система сохранений (scr_save_system)

### Спецификации сохранения
- Должен сохраняться `global.game_state`
- Должна сохраняться структура `global.game_progress`:
  - Статусы уровней (completed, lever_pulled)
  - Количество золота (gold)
  - Предметы (items)
  - Статусы мини-игр (mechanic_training_completed, archivist_quiz_completed, elder_trial_completed)
- Файл сохранения должен называться `adventure_puzzle_save.sav`
- Данные должны быть сериализованы в формате JSON

### Спецификации загрузки
- Из файла сохранения должны восстанавливаться все данные прогресса
- `global.game_state` должен измениться на сохраненное значение
- Структура `global.game_progress` должна полностью восстановиться
- Список предметов должен быть восстановлен из сохранения

### Спецификации сброса
- Файл сохранения должен быть удален
- `global.game_progress` должен быть сброшен к начальным значениям
- Количество золота должно стать 100
- Все уровни должны быть не завершены и рычаги не активированы
- Статусы мини-игр должны быть `false`

## Аудио-менеджер (scr_audio_manager)

### Спецификации воспроизведения
- `play_music()` должен начинать воспроизведение музыкального файла
- `play_sound()` должен начинать воспроизведение звукового эффекта
- Должна поддерживаться настройка громкости для музыки и SFX отдельно
- `stop_music()` должен останавливать текущую проигрываемую музыку
- При недоступности аудио-системы все функции должны работать без ошибок (fail-safe)

### Спецификации громкости
- `master_volume` управляет общей громкостью
- `music_volume` управляет громкостью музыки
- `sfx_volume` управляет громкостью звуковых эффектов
- Финальная громкость рассчитывается как `base_volume * master_volume * specific_volume`
- Значения громкости должны быть в пределах от 0 до 1

## UI-менеджер (scr_ui_manager)

### Спецификации уведомлений
- `show_notification()` должен добавлять уведомление в очередь
- Уведомления должны отображаться в верхней части экрана
- Уведомления должны исчезать через заданное время
- Должна поддерживаться возможность отображения нескольких уведомлений одновременно

### Спецификации прогресса
- `draw_expedition_progress()` должен визуально отображать прогресс экспедиции
- Должна отображаться информация о числе пройденных уровней
- Должна отображаться текущая цель экспедиции
- Должна быть визуальная индикация открытых уровней

### Спецификации диалогов
- `show_dialog()` должен отображать диалоговое окно с текстом и вариантами ответов
- Должна поддерживаться навигация по опциям с помощью стрелок
- Выбор опции должен происходить по нажатию Enter
- Диалог должен закрываться по нажатию Escape

## Игровой контроллер (scr_game_controller)

### Спецификации состояний
- `game_state` может быть: `"menu"`, `"town"`, `"playing_level_1-6"`, `"victory"`
- При смене состояния должна происходить соответствующая реакция (например, смена музыки)
- `set_game_state()` должен проверять корректность переданного состояния

### Спецификации прогресса
- `complete_level(level_num)` должен устанавливать `completed = true` для указанного уровня
- `set_lever_pulled(level_num, pulled)` должен устанавливать `lever_pulled = true` только если уровень завершён
- `is_level_unlocked(level_num)` должен возвращать `true` только если предыдущий уровень активировал рычаг
- `apply_progress_update()` должен корректно обновлять статусы мини-игр и выдавать награды

## Правила тестирования

### Правило 1: Автоматизация
- Все тесты должны быть автоматизируемыми
- Тесты должны возвращать логическое значение успеха/неудачи
- Тесты не должны требовать пользовательского ввода

### Правило 2: Изолированность
- Каждый тест должен быть независим от других
- Тесты должны устанавливать начальное состояние перед выполнением
- Тесты не должны полагаться на результаты других тестов

### Правило 3: Полнота
- Тесты должны охватывать все публичные методы и свойства
- Тесты должны проверять как нормальные, так и крайние случаи
- Тесты должны проверять как положительные, так и отрицательные сценарии

### Правило 4: Читаемость
- Имена тестов должны четко описывать тестируемую функциональность
- В описании теста должна быть указана цель, шаги и ожидаемый результат
- Тесты должны быть документированы и понятны другим разработчикам

## Статусы тестов

- `PASS` - тест пройден успешно
- `FAIL` - тест не прошел, есть ошибка
- `BLOCKED` - тест не может быть выполнен из-за зависимостей
- `SKIP` - тест временно пропущен
- `TODO` - тест еще не реализован
