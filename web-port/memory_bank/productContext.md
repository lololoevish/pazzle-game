# Product Context

## Назначение продукта

Проект дает runnable web-port приключенческой puzzle-игры, чтобы игру можно было запускать и проверять в браузере без установки GameMaker Studio 2.

## Пользовательские проблемы

- Игроку нужен простой запуск через браузер и понятное клавиатурное управление.
- Разработчику или автору контента нужна возможность заменить визуальные заглушки на собственные спрайты без правки TypeScript-кода.
- Прогресс прохождения должен сохраняться между сессиями.

## Основные пользовательские сценарии

1. Игрок запускает игру, начинает новую игру или продолжает сохранение.
2. Игрок перемещается по городу-хабу, получает награды от NPC и входит в пещеры.
3. Игрок проходит серию пещер с разными механиками пазлов.
4. После завершения 24 пещер игрок видит экран победы.
5. Автор заменяет спрайты через `public/sprites/manifest.json` и перезагружает игру.
