# Active Context

## Текущий фокус

Устранение зависания при запуске игры (черный экран на старте) из-за некорректной обработки асинхронного `preload` в Phaser 3.

## Активные решения

- Папка `memory_bank/` отсутствовала и создана заново.
- Так как `docs/README.md` отсутствует, первичная синхронизация выполнена по `README.md` и ограниченному обзору кода в `src/`.
- `## Project Deliverables` создан как каноническая markdown-таблица с весами, сумма которых равна 100.
- Phaser Scale Manager переведен с `RESIZE` на `FIT`, чтобы базовый мир 800x600 масштабировался целиком и сохранял aspect ratio на разных мониторах.
- CSS canvas больше не растягивает игру принудительно на `100vw x 100vh`; ограничение идет через `max-width`/`max-height` с поддержкой `100dvh`.
- Переработан `BootScene.ts`: убран асинхронный `preload`, ломавший жизненный цикл Phaser 3. Манифест теперь безопасно загружается как встроенный JSON-ресурс, а постановка файлов в очередь и запуск загрузчика вынесены в `create` с проверкой на наличие изображений. Это полностью решило проблему бесконечной загрузки (черного экрана) на старте.

## Приоритеты

1. Поддерживать `memory_bank/projectbrief.md` как канонический источник процента готовности проекта.
2. При изменениях scope немедленно обновлять deliverables и проверять сумму Weight.
3. При архитектурных изменениях синхронизировать документацию проекта.
