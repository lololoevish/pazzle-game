# System Patterns

## Архитектура

Игра построена как клиентское Phaser 3 приложение на TypeScript. Точка входа `src/main.ts` создает `Phaser.Game` и регистрирует сцены.

## Сцены

- `BootScene` — загрузка manifest спрайтов, постановка ресурсов в очередь и переход в меню.
- `MenuScene` — старт новой игры, продолжение сохранения и переход к справке по спрайтам.
- `SpriteHelpScene` — внутриигровая инструкция по подключению пользовательских PNG.
- `TownScene` — город-хаб, NPC, награды, препятствия, управление персонажем и вход в пещеры.
- `CaveScene` — прохождение пещер с puzzle-механиками, прогрессом уровней и рычагами завершения.
- `VictoryScene` — финальный экран после завершения экспедиции.

## Системные паттерны

- Состояние прохождения нормализуется через `GameState` перед сохранением и чтением.
- Сохранение использует `localStorage` через `SaveSystem`.
- Визуальные ассеты подменяемы через manifest; отсутствующие ассеты компенсируются fallback-текстурами.
- UI-хелперы вынесены в `systems/UiSystem.ts`, диалоги — в `systems/DialogueSystem.ts`.
- Игровой мир остается в базовом разрешении 800x600, а Phaser Scale Manager в режиме `FIT` вписывает canvas в монитор без растяжения и обрезания.
