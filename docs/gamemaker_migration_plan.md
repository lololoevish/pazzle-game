# План миграции на GameMaker Studio 2

## Назначение документа

Данный документ описывает поэтапный процесс миграции функционала "Adventure Puzzle Game" из Rust-версии в GameMaker Studio 2.

## Объективы миграции

1. Перенести все игровые механики из Rust-версии в GameMaker 2
2. Сохранить игровой баланс и сложность
3. Обеспечить аналогичный пользовательский опыт
4. Подготовить проект для дальнейшей разработки в GameMaker

## Фазы миграции

### Фаза 1: Подготовка проекта (Week 1)

#### Задачи:
1. Установить GameMaker Studio 2 (версия 2022.6 или новее)
2. Создать новый проект `adventure_puzzle`
3. Настроить основные конфигурации
4. Создать структуру папок:
   - `Objects`
   - `Rooms`
   - `Sprites`
   - `Sounds`
   - `Scripts`
   - `Fonts`

#### Ожидаемые результаты:
- Созданный проект GameMaker
- Настроенная структура папок
- Установленная основа архитектуры

### Фаза 2: Создание базовой архитектуры (Week 2)

#### Задачи:
1. Создать основные объекты:
   - `obj_game_manager` - управление состоянием игры
   - `obj_player` - игрок и его поведение
   - `obj_scene_manager` - управление сценами
   - `obj_ui_manager` - интерфейс

2. Реализовать основные скрипты:
   - `scr_game_state` - система состояний
   - `scr_save_system` - система сохранений
   - `scr_audio_manager` - аудио
   - `scr_ui_manager` - UI система

3. Настроить систему сохранений

#### Ожидаемые результаты:
- Базовая архитектура проекта
- Система состояний игры
- Базовая система сохранений

### Фаза 3: Реализация основных комнат (Week 3)

#### Задачи:
1. Создать основные комнаты:
   - `rm_menu` - главное меню
   - `rm_hub_city` - город-хаб
   - `rm_level_1_maze` - лабиринт
   - `rm_level_2_word_search` - поиск слов
   - `rm_level_3_memory_patterns` - память/паттерны
   - `rm_level_4_memory_puzzle` - головоломка на память
   - `rm_level_5_platformer` - платформер
   - `rm_level_6_final_challenge` - финальный уровень
   - `rm_victory` - финальная сцена

2. Настроить базовую структуру комнат
3. Создать объекты-заглушки для игровых элементов

#### Ожидаемые результаты:
- Все комнаты созданы с базовой структурой
- Возможность перехода между комнатами
- Объекты-заглушки для всех игровых элементов

### Фаза 4: Миграция системы меню и хаба (Week 4)

#### Задачи:
1. Реализовать `rm_menu` с полной функциональностью:
   - Анимации
   - Навигация
   - Сохранение прогресса
   - Сброс игры

2. Реализовать `rm_hub_city`:
   - Движение игрока
   - Взаимодействие с NPC
   - Переходы к уровням
   - Система прогресса

3. Создать NPC объекты:
   - `obj_npc_base` - базовый класс
   - `obj_npc_mechanic` - механик Роан
   - `obj_npc_archivist` - архивариус Тель
   - `obj_npc_elder` - староста Иар

#### Ожидаемые результаты:
- Полнофункциональное главное меню
- Работающий хаб-город
- Взаимодействие с NPC

### Фаза 5: Миграция головоломки "Лабиринт" (Week 5)

#### Задачи:
1. Реализовать `rm_level_1_maze`:
   - Генерация лабиринта
   - Система движения (скольжение)
   - Обнаружение столкновений
   - Система решения головоломки
   - Активация рычага

2. Настроить визуализацию
3. Создать систему подсказок

#### Ожидаемые результаты:
- Полнофункциональный уровень "Лабиринт"
- Работающая система решения
- Активация рычага при прохождении

### Фаза 6: Миграция головоломки "Поиск слов" (Week 6)

#### Задачи:
1. Реализовать `rm_level_2_word_search`:
   - Генерация сетки слов
   - Система поиска слов
   - Две фазы: поиск слов и memory-головоломка
   - Проверка правильности решения

2. Интегрировать Russian word list
3. Настроить визуализацию

#### Ожидаемые результаты:
- Полнофункциональный уровень "Поиск слов"
- Работающая двухфазная головоломка
- Активация рычага при прохождении

### Фаза 7: Миграция головоломки "Память/Паттерны" (Week 7)

#### Задачи:
1. Реализовать `rm_level_3_memory_patterns`:
   - Система "Simon Says"
   - Генерация последовательностей
   - Система увеличения сложности
   - Проверка правильности ответов

2. Настроить аудиообратную связь
3. Создать систему прогрессии сложности

#### Ожидаемые результаты:
- Полнофункциональный уровень "Память/Паттерны"
- Работающая система последовательностей
- Активация рычага при прохождении

### Фаза 8: Миграция головоломки "Память" (Week 8)

#### Задачи:
1. Реализовать `rm_level_4_memory_puzzle`:
   - Система карточек
   - Поиск пар
   - Таймер (опционально)
   - Проверка совпадений

2. Настроить анимации переворота
3. Интегрировать систему прогресса

#### Ожидаемые результаты:
- Полнофункциональный уровень "Память"
- Работающая система карточек
- Активация рычага при прохождении

### Фаза 9: Миграция головоломки "Платформер" (Week 9)

#### Задачи:
1. Реализовать `rm_level_5_platformer`:
   - Физика прыжков
   - Система препятствий
   - Сбор кристаллов/ключей
   - Победные условия

2. Настроить управление
3. Создать систему контрольных точек

#### Ожидаемые результаты:
- Полнофункциональный платформер
- Работающая физика и управление
- Активация рычага при прохождении

### Фаза 10: Миграция финального уровня (Week 10)

#### Задачи:
1. Реализовать `rm_level_6_final_challenge`:
   - Таймер

   - Финальное испытание
   - Переход к победной сцене

2. Настроить драматическую подачу
3. Интегрировать аудио и визуальные эффекты

#### Ожидаемые результаты:

- Работающее финальное испытание
- Переход к победной сцене

### Фаза 11: Миграция NPC мини-игр (Week 11)

#### Задачи:
1. Реализовать мини-игры для NPC:
   - Калибровка механика Роана
   - Викторина архивариуса Теля

3. Интегрировать в систему прогресса

#### Ожидаемые результаты:
- Полнофункциональные мини-игры

- Интеграция с основным прогрессом

### Фаза 12: Интеграция аудио и визуальных эффектов (Week 12)

#### Задачи:
1. Реализовать полную аудио-систему:
   - Фоновая музыка для всех сцен
   - Звуковые эффекты

2. Добавить визуальные эффекты:
   - Анимации

#### Ожидаемые результаты:

## Технические требования

### Архитектурные паттерны:
- Использование объектно-ориентированного подхода
- Создание системы наследования для объектов
- Разделение логики и представления
- Модульная структура кода

### Производительность:
- Поддержка 60 FPS на целевых платформах
- Эффективное управление памятью
- Оптимизация ресурсов

### Совместимость:
- Сохранение данных между Rust и GMS2 версиями (при возможности)
- Поддержка тех же разрешений экрана (800x600)
- Аналогичные системные требования

## Риски и ограничения

### Технические риски:
1. Разница в производительности между движками
2. Необходимость адаптации графических ресурсов
3. Особенности GML по сравнению с Rust

### Функциональные ограничения:
1. Некоторые низкоуровневые оптимизации могут быть недоступны
2. Ограничения GameMaker могут повлиять на реализацию некоторых механик
3. Необходимость лицензии для разработки

## Критерии успеха

1. Все игровые механики полностью реализованы
2. Производительность соответствует требованиям
3. Все уровни проходимы и соответствуют оригиналу
4. Система сохранений работает корректно
5. Игра полностью функциональна и не содержит критических багов
