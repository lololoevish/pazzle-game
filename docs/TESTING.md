# Документ тестирования GameMaker 2 проекта

## Общая информация

Этот документ описывает сценарии интеграционного тестирования для проверки корректности функциональности GameMaker 2 версии "Adventure Puzzle Game".

## Сценарии тестирования

### 1. Система состояний игры

#### Тест 1.1: Главное меню
- **Предусловие**: Игра запущена
- **Шаги**:
  1. Проверить отображение главного меню
  2. Проверить навигацию по меню (стрелки/WASD + Enter)
  3. Проверить возможность запуска новой игры
  4. Проверить возможность продолжения сохранённой игры
- **Ожидаемый результат**: Меню отображается корректно, навигация работает

#### Тест 1.2: Переходы между экранами
- **Предусловие**: Игра запущена
- **Шаги**:
  1. Начать новую игру
  2. Проверить переход в хаб-город
  3. Проверить возврат в меню
- **Ожидаемый результат**: Переходы происходят без ошибок

### 2. Система уровней

#### Тест 2.1: Доступность уровней
- **Предусловие**: Игрок находится в хаб-городе
- **Шаги**:
  1. Проверить доступность первого уровня
  2. Пройти первый уровень
  3. Проверить открытие второго уровня
- **Ожидаемый результат**: Уровни открываются последовательно

#### Тест 2.2: Функциональность головоломок
- **Предусловие**: Игрок находится на уровне с головоломкой
- **Шаги**:
  1. Проверить функциональность головоломки
  2. Решить головоломку
  3. Проверить возможность активации рычага
- **Ожидаемый результат**: Головоломка решается, рычаг активируется

### 3. Система сохранений

#### Тест 3.1: Сохранение прогресса
- **Предусловие**: Игрок достиг определённого прогресса
- **Шаги**:
  1. Сохранить игру
  2. Выйти из игры
  3. Запустить игру заново
  4. Выбрать "Продолжить"
- **Ожидаемый результат**: Прогресс сохранён и загружен корректно

#### Тест 3.2: Автосохранение
- **Предусловие**: Игрок проходит уровни
- **Шаги**:
  1. Пройти несколько уровней
  2. Выйти из игры
  3. Запустить игру
- **Ожидаемый результат**: Игра автоматически сохраняет прогресс

### 4. Система NPC и мини-игр

#### Тест 4.1: Взаимодействие с NPC
- **Предусловие**: Игрок находится в хаб-городе
- **Шаги**:
  1. Подойти к NPC
  2. Активировать диалог
  3. Пройти мини-игру
- **Ожидаемый результат**: Диалог и мини-игра работают корректно

#### Тест 4.2: Награды за мини-игры
- **Предусловие**: Игрок завершает мини-игру
- **Шаги**:
  1. Завершить мини-игру
  2. Проверить получение награды
  3. Проверить отображение награды в инвентаре
- **Ожидаемый результат**: Награда начисляется и отображается

### 5. Система аудио

#### Тест 5.1: Воспроизведение музыки
- **Предусловие**: Игра запущена
- **Шаги**:
  1. Перейти в разные зоны (меню, хаб, уровни)
  2. Проверить воспроизведение соответствующей музыки
- **Ожидаемый результат**: Музыка воспроизводится корректно

#### Тест 5.2: Звуковые эффекты
- **Предусловие**: Игрок взаимодействует с объектами
- **Шаги**:
  1. Выполнить действия (движение, подбор предметов, нажатие кнопок)
  2. Проверить наличие звуковых эффектов
- **Ожидаемый результат**: Звуковые эффекты воспроизводятся

### 6. Система UI

#### Тест 6.1: Отображение интерфейса
- **Предусловие**: Игрок находится в разных состояниях
- **Шаги**:
  1. Проверить отображение UI в меню
  2. Проверить отображение UI в хабе
  3. Проверить отображение UI на уровнях
- **Ожидаемый результат**: UI отображается корректно в зависимости от состояния

#### Тест 6.2: Реагирование интерфейса
- **Предусловие**: Игрок взаимодействует с UI
- **Шаги**:
  1. Нажать на элементы интерфейса
  2. Проверить реакцию на действия
- **Ожидаемый результат**: UI реагирует на действия игрока

## Требования к тестированию

1. Тестирование должно проводиться на всех поддерживаемых платформах
2. Все сценарии должны выполняться без ошибок
3. Производительность должна оставаться стабильной (60 FPS)
4. Сохранения должны работать корректно при всех сценариях

## Отчет о тестировании

После выполнения тестов необходимо заполнить отчет с указанием:
- Протестированных сценариев
- Обнаруженных ошибок
- Рекомендаций по улучшению
- Общего состояния проекта
