# Архитектурный документ GameMaker Studio 2 проекта

## Обзор

Данный документ описывает архитектуру проекта "Adventure Puzzle Game" для GameMaker Studio 2. Он определяет структуру проекта, основные объекты, комнаты, скрипты и архитектурные паттерны, которые будут использоваться при миграции из Rust-версии.

## Структура проекта

### Объекты (Objects)

#### 1. Управляющие объекты
- `obj_game_manager`: Основной управляющий объект, существует на всех комнатах
  - Управляет общим состоянием игры
  - Контролирует переходы между комнатами
  - Управляет системой сохранений
  - Управляет аудио-сессией
  
- `obj_scene_manager`: Управляет логикой текущей сцены
  - Следит за прогрессом на уровне
  - Активирует события при выполнении условий
  - Управляет состоянием головоломки
  
- `obj_ui_manager`: Управляет интерфейсом
  - Отображает здоровье, очки, таймеры
  - Обрабатывает пользовательский ввод
  - Управляет экранами паузы и меню

#### 2. Игрок и взаимодействие
- `obj_player`: Основной объект игрока
  - Обработка ввода
  - Движение и физика
  - Взаимодействие с другими объектами
  - Анимации
  
- `obj_interactable`: Абстрактный объект для взаимодействия
  - Дочерние объекты: `obj_lever`, `obj_door`, `obj_npc`, `obj_item`
  - Определяет базовую логику взаимодействия

#### 3. Специфичные для уровней объекты
- `obj_maze_generator`: Генератор лабиринта
  - Создает структуру лабиринта
  - Проверяет решение
  
- `obj_word_grid`: Объект сетки поиска слов
  - Управляет расположением букв
  - Проверяет найденные слова
  
- `obj_simon_block`: Блоки для паттернов "Simon Says"
  - Анимация и световые эффекты
  - Генерация последовательности
  
- `obj_memory_card`: Карточки для игры на память
  - Состояние: открытая/закрытая
  - Изображение/значение карточки
  
- `obj_platform_player`: Улучшенный игрок для платформера
  - Физика прыжков
  - Взаимодействие с платформами и триггерами
  
- `obj_final_timer`: Таймер для финального уровня
  - Обратный отсчет
  - Обработка истечения времени

#### 4. NPC и мини-игры
- `obj_npc_base`: Базовый класс для всех NPC
  - Обработка взаимодействия
  - Управление диалогами
  
- `obj_npc_mini_game`: NPC с мини-играми
  - Дочерние: `obj_npc_mechanic`, `obj_npc_archivist`, `obj_npc_elder`
  - Запуск мини-игр
  - Выдача наград

### Комнаты (Rooms)

#### 1. Основные комнаты
- `rm_menu`: Главное меню
  - Анимации и визуальные эффекты
  - Навигация: начать игру, загрузить, настройки
  
- `rm_hub_city`: Город-хаб
  - Свободное перемещение игрока
  - Взаимодействие с NPC
  - Переходы к уровням
  - Отображение прогресса
  
- `rm_victory`: Финальная сцена
  - Победная анимация
  - Статистика прохождения
  - Возврат в меню

#### 2. Уровни
- `rm_level_1_maze`: Уровень лабиринта
  - Генерируемая структура лабиринта
  - Игрок с возможностью скольжения
  - Рычаг для активации при решении
  
- `rm_level_2_word_search`: Уровень поиска слов
  - Сетка букв
  - Интерфейс для выделения слов
  - Две фазы решения
  
- `rm_level_3_memory_patterns`: Уровень памяти/паттернов
  - Интерактивные блоки
  - Система последовательностей
  - Прогрессия сложности
  
- `rm_level_4_memory_puzzle`: Уровень игры на память
  - Сетка карточек
  - Система переворота и сопоставления
  - Таймер (при необходимости)
  
- `rm_level_5_platformer`: Платформер
  - Физика прыжков
  - Препятствия и контрольные точки
  - Сбор элементов
  
- `rm_level_6_final_challenge`: Финальный уровень
  - Комбинация предыдущих механик
  - Таймер
  - Сложные головоломки

### Скрипты (Scripts)

#### 1. Системные скрипты
- `scr_game_state`: Менеджер состояния игры
  - Состояния: меню, игра, пауза, загрузка
  - Переключение между состояниями
  
- `scr_save_system`: Система сохранений
  - Сохранение/загрузка прогресса
  - Уровень прогресса, очки, настройки
  
- `scr_audio_manager`: Управление аудио
  - Воспроизведение музыки
  - Воспроизведение звуковых эффектов
  - Управление громкостью
  
- `scr_ui_manager`: Система интерфейса
  - Отрисовка UI элементов
  - Обработка ввода от пользователя
  
- `scr_utilities`: Вспомогательные функции
  - Математические утилиты
  - Работа со строками
  - Конвертация типов

#### 2. Уровневые скрипты
- `scr_maze_generation`: Генерация лабиринта
  - Алгоритмы создания лабиринта
  - Проверка проходимости
  
- `scr_word_search_logic`: Логика поиска слов
  - Генерация сетки
  - Проверка слов
  - Управление двумя фазами игры
  
- `scr_simon_says_logic`: Логика "Simon Says"
  - Генерация последовательностей
  - Проверка ответов
  - Увеличение сложности
  
- `scr_memory_game_logic`: Логика игры на память
  - Создание пар карточек
  - Проверка совпадений
  - Система счета
  
- `scr_platformer_physics`: Физика платформера
  - Гравитация
  - Столкновения
  - Движение и прыжки

#### 3. Скрипты NPC и диалогов
- `scr_dialog_system`: Система диалогов
  - Управление текстом диалогов
  - Порядок фраз
  - Реакция на выбор игрока
  
- `scr_npc_behavior`: Поведение NPC
  - Анимации
  - Реакции на взаимодействие
  - Состояния NPC

## Архитектурные паттерны

### 1. Состояния игры (Game States)
Используется для управления различными состояниями игры:
- Меню
- Игра
- Пауза
- Диалог
- Мини-игра
- Сохранение/загрузка

### 2. Синглтон-менеджеры
- `global.game_manager` - глобальный доступ к управляющим функциям
- `global.save_data` - глобальный доступ к данным сохранения
- `global.audio_manager` - глобальный доступ к аудио функциям

### 3. Наследование объектов
- Базовые классы определяют общее поведение
- Дочерние классы переопределяют специфичное поведение
- Упрощает поддержку и расширение функционала

### 4. События и подписки
- Использование GML event system для взаимодействия между объектами
- Сигналы об изменениях состояния
- Система уведомлений о событиях
