# Техническое задание: Миграция игры на GameMaker 2

## Введение
Цель: Перенос функционала приключенческой 2D игры с головоломками из Rust-версии в GameMaker Studio 2 с сохранением всех игровых механик и возможностью дальнейшей разработки.

## Требования к функционалу

### 1. Основная архитектура системы
- Использовать объектно-ориентированный подход GameMaker
- Создать систему состояний игры (меню, хаб, уровни, пауза)
- Реализовать систему сохранений и загрузок
- Настроить систему прогресса между уровнями

### 2. Главное меню
- Анимированное главное меню с кнопками
- Система навигации (стрелки/WASD + Enter)
- Звуковое сопровождение

### 3. Система уровней
- 6 уникальных уровней-головоломок:
  - Лабиринт (с процедурной генерацией)
  - Поиск слов (с русскими словами)
  - Память/Паттерны (Simon Says)
  - Головоломка на память
  - Платформер с прыжками
  - Финальный уровень с таймером
- Система переходов между уровнями
- Система автосохранений

### 4. Хаб-город
- Центральная локация для выбора уровней
- NPC с диалогами и мини-играми
- Система инвентаря (золото, предметы, награды)
- Возможность возвращаться к завершенным уровням

### 5. NPC и мини-игры
- Интерактивные NPC с диалоговыми системами
- Три типа мини-игр:
  - Угадай число
  - Камень-ножницы-бумага
  - Викторина
- Система наград и достижений

### 6. Система инвентаря
- Система сбора и использования предметов
- Визуализация инвентаря
- Система валюты (золото)

### 7. Система диалогов
- Система реплик с выбором
- Портреты персонажей
- Историческое дерево диалогов

### 8. Звуковая система
- Фоновая музыка для разных зон
- Звуковые эффекты для взаимодействий
- Система настройки громкости

### 9. Адаптация графики
- Импорт спрайтов из ресурсов
- Анимация персонажей и объектов
- Адаптация интерфейса под разрешение

## Технические требования

### Архитектура GameMaker проекта
- Objects: 
  - obj_player - главный персонаж
  - obj_level_controller - управление уровнем
  - obj_dialog_system - система диалогов
  - obj_npc_base - базовый класс NPC
  - obj_inventory_system - система инвентаря
  - obj_save_manager - управление сохранениями
  - obj_ui_manager - интерфейс
  - obj_game_manager - основное состояние игры

- Rooms:
  - rm_menu - главное меню
  - rm_hub_city - город-хаб
  - rm_level_1-6 - игровые уровни
  - rm_pause - экран паузы

- Scripts:
  - scr_game_state() - управление состоянием игры
  - scr_level_transition() - переходы между уровнями
  - scr_save_load() - система сохранений
  - scr_dialog_manager() - управление диалогами
  - scr_inventory_system() - система инвентаря
  - scr_npc_interactions() - взаимодействия с NPC
  - scr_minigame_logic() - логика мини-игр
  - scr_audio_manager() - управление аудио

### Система навигации
- Клавиатурное управление: стрелки/WASD для движения, Enter для подтверждения
- Мышиное управление для интерфейса
- ESC для возврата/паузы

### Система сохранений
- Сохранение прогресса между сессиями
- Сохранение состояния уровней
- Сохранение инвентаря и наград
- Поддержка нескольких слотов сохранения

## Требования к производительности
- Стабильные 60 FPS
- Оптимизация ресурсов
- Минимальное время загрузки уровней

## Этапы реализации

### Этап 1: Базовая архитектура
- Создание основного меню
- Реализация системы состояний
- Настройка основных объектов

- Реализация комнаты хаба
- Добавление NPC
- Создание системы взаимодействия

### Этап 3: Уровни 1-2
- Реализация первых двух головоломок
- Тестирование системы переходов

### Этап 4: Уровни 3-4
- Реализация третьей и четвертой головоломок
- Настройка системы сохранений

### Этап 5: Уровни 5-6
- Реализация пятой и шестой головоломок
- Добавление таймера и финального испытания

### Этап 6: NPC и мини-игры
- Добавление интерактивных NPC
- Реализация трех типов мини-игр

### Этап 7: Звук и визуальные эффекты
- Добавление аудио
- Настройка системной графики
- Финальная полировка

### Этап 8: Тестирование и доводка
- Тестирование всех систем
- Оптимизация производительности
