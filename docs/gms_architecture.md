# Архитектура GameMaker 2 проекта "Adventure Puzzle Game"

## Общая структура проекта

### 1. Основные объекты (Objects)

#### Системные объекты:
- `obj_game_manager`
  - Управляет глобальным состоянием игры
  - Инициализирует систему сохранений
  - Координирует переходы между состояниями
  
- `obj_scene_manager`
  - Управляет сценами и комнатами
  - Обрабатывает переходы между уровнями
  - Управляет памятью и ресурсами

- `obj_save_manager`
  - Обрабатывает сохранение и загрузку данных
  - Хранит прогресс, инвентарь, статус уровней
  - Работает с GML структурами данных

#### Игровые объекты:
- `obj_player`
  - Управление персонажем
  - Обработка ввода
  - Коллизии и физика
  
- `obj_npc_base`
  - Базовый класс для всех NPC
  - Содержит диалоговую систему
  - Обработка взаимодействий
  
- `obj_level_controller`
  - Управление логикой уровня
  - Система завершения головоломок
  - Связь с менеджером сцен

#### UI объекты:
- `obj_ui_manager`
  - Основной интерфейс
  - Меню и элементы навигации
  - Обработка UI событий

- `obj_inventory_system`
  - Управление инвентарем
  - Система предметов и наград
  - Отображение UI инвентаря

- `obj_dialog_system`
  - Система диалогов
  - Управление репликами
  - Показ портретов персонажей

#### Системные компоненты:
- `obj_audio_manager`
  - Управление воспроизведением музыки
  - Система звуковых эффектов
  - Настройка громкости

- `obj_particle_system`
  - Система эффектов
  - Визуальная обратная связь
  - Анимация элементов интерфейса

### 2. Комнаты (Rooms)

#### Основные комнаты:
- `rm_menu`
  - Главное меню игры
  - Навигация между основными сценами
  - Настройки и информация

- `rm_hub_city`
  - Центральная локация
  - Переходы к уровням
  - Взаимодействие с NPC
  - Магазин и инвентарь

#### Уровни:
- `rm_level_1_maze`
  - Генерируемый лабиринт
  - Система перемещения
  - Побочные объекты

- `rm_level_2_word_search`
  - Система поиска слов
  - Алгоритмы генерации сетки
  - Система проверки ответов

- `rm_level_3_memory_patterns`
  - Игра "Simon Says"
  - Генерация последовательностей
  - Система увеличения сложности

- `rm_level_4_memory_puzzle`
  - Игра на память
  - Система карточек
  - Проверка совпадений

- `rm_level_5_platformer`
  - Платформенная головоломка
  - Физика прыжков
  - Система препятствий

- `rm_level_6_final_challenge`
  - Финальный уровень
  - Таймер и ограничения
  - Система победы

#### Вспомогательные комнаты:
- `rm_pause`
  - Экран паузы
  - Настройки и навигация

- `rm_settings`
  - Системные настройки
  - Звуки, графика, управление

### 3. Скрипты (Scripts)

#### Основные скрипты:
- `scr_game_state()`
  - Управление глобальным состоянием
  - Перечисление состояний: MENU, HUB, LEVEL, PAUSE, DIALOG, MINIGAME

- `scr_level_transition(room_id, transition_type)`
  - Система переходов между комнатами
  - Анимации переходов
  - Сохранение состояния перед переходом

- `scr_save_load(operation, slot)`
  - Унифицированная система сохранений
  - Операции: save, load, delete, exists
  - Поддержка нескольких слотов

- `scr_dialog_manager(character_id, dialog_tree)`
  - Управление диалоговыми деревьями
  - Система реплик и выборов
  - Интеграция с инвентарем

#### Логические скрипты:
- `scr_puzzle_logic(puzzle_type, params)`
  - Обработка логики головоломок
  - Проверка условий победы
  - Управление состоянием уровня

- `scr_npc_behavior(npc_id, interaction_type)`
  - Поведение NPC
  - Система мини-игр
  - Обработка наград

- `scr_inventory_operations(action, item_id, quantity)`
  - Операции с инвентарем
  - Система получения и расхода предметов
  - Проверка условий использования

#### Вспомогательные скрипты:
- `scr_math_utils()`
  - Математические утилиты
  - Генерация случайных чисел
  - Работа с векторами

- `scr_string_utils()`
  - Утилиты для работы со строками
  - Локализация
  - Поиск слов

- `scr_pathfinding()`
  - Система поиска пути
  - Алгоритмы обхода
  - Обработка препятствий

### 4. События и управление состоянием

#### Система событий:
- Использование GML событий (Create, Step, Draw, Collision)
- Собственная система сообщений через переменные
- Система callback функций

#### Управление состоянием:
- Глобальные переменные для состояния игры
- Структуры данных для хранения прогресса
- Система сигналов между объектами

### 5. Система сохранений

#### Структура сохранения:
```gml
struct save_data {
    game_version,
    timestamp,
    player_progress,
    inventory,
    level_statuses,
    settings,
    dialog_progress
}
```

#### Форматы:
- Использование JSON для сохранений
- Шифрование чувствительных данных
- Резервные копии

### 6. Архитектурные паттерны

#### Синглтон объекты:
- `obj_game_manager` как глобальный синглтон
- Централизованное управление состоянием

#### Компонентная архитектура:
- Базовые классы с наследованием
- Reuse кода через скрипты

#### Состояния (State Pattern):
- Основные состояния игры
- Четкое разделение логики по состояниям

#### Управление памятью:
- Удаление неиспользуемых объектов

- Оптимизация рендеринга

#### Система загрузки:
- Асинхронная загрузка ресурсов
- Прогресс-бар для длительных операций
- Кэширование часто используемых данных

#### Система логирования:
