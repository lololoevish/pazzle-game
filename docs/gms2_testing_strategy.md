# Документ тестирования GameMaker 2 проекта

## Обзор

Этот документ описывает стратегию и процессы тестирования для GameMaker Studio 2 версии "Adventure Puzzle Game". Включает в себя подходы к модульному, интеграционному и функциональному тестированию.

## Стратегия тестирования

### 1. Модульное тестирование (Unit Testing)
Цель: Проверка отдельных компонентов системы на корректность работы

**Объекты для тестирования:**
- `obj_game_manager`
- `obj_scene_manager`
- `obj_puzzle_controller`
- Специфичные объекты головоломок
- Вспомогательные скрипты

**Что тестируем:**
- Корректность обработки входных данных
- Правильность возвращаемых значений
- Состояния объектов
- Взаимодействие компонентов

### 2. Интеграционное тестирование (Integration Testing)
Цель: Проверка взаимодействия между различными системами и компонентами

**Сценарии:**
- Взаимодействие между менеджерами состояний
- Связь между системой прогресса и объектами уровней
- Работа сохранений с различными системами

### 3. Функциональное тестирование (Functional Testing)
Цель: Проверка соответствия функциональности требованиям

**Тестирование по сценариям:**
- Полный прогон игры от меню до победы
- Проверка всех головоломок
- Проверка мини-игр у NPC
- Проверка системы сохранений

## План тестирования

### Этап 1: Модульное тестирование отдельных компонентов

#### 1.1. Система состояний
**Объект:** `obj_game_manager`

**Тесты:**
- [ ] Корректная установка глобального состояния (menu, town, playing, victory)
- [ ] Сохранение и загрузка состояния игры
- [ ] Обработка переходов между состояниями

**Пример теста:**
```gml
// Тест установки состояния
var initial_state = global.game_state;
obj_game_manager.set_state(gs_playing);
assert_equal(global.game_state, gs_playing, "Состояние не изменилось правильно");

obj_game_manager.set_state(initial_state);
```

#### 1.2. Система прогресса
**Объект:** `obj_game_manager`

**Тесты:**
- [ ] Инициализация структуры прогресса по умолчанию
- [ ] Правильное завершение уровня
- [ ] Установка состояния рычага
- [ ] Применение обновлений прогресса

#### 1.3. Логика головоломок
**Объекты:** `obj_maze_puzzle`, `obj_word_search_puzzle`, `obj_pattern_puzzle`, и т.д.

**Тесты:**
- [ ] Правильная генерация головоломок
- [ ] Корректная обработка ввода
- [ ] Условия завершения головоломки
- [ ] Проверка корректности решений

### Этап 2: Интеграционное тестирование

#### 2.1. Система сохранений
**Компоненты:** `obj_game_manager`, `scr_save_system`

**Тесты:**
- [ ] Сохранение и загрузка прогресса уровней
- [ ] Сохранение и загрузка состояния мини-игр NPC
- [ ] Совместимость с Rust-версией (если применимо)

#### 2.2. Система взаимодействия
**Компоненты:** `obj_player`, `obj_interactable`, `obj_scene_manager`

**Тесты:**
- [ ] Правильное взаимодействие с объектами в сценах
- [ ] Обработка переходов между комнатами
- [ ] Передача состояний между сценами

### Этап 3: Функциональное тестирование

#### 3.1. Полный игровой цикл
**Сценарии:**
- [ ] Начало новой игры → меню
- [ ] Навигация в город → взаимодействие с NPC → запуск мини-игр
- [ ] Переход к уровням → прохождение головоломок
- [ ] Активация рычагов → открытие проходов
- [ ] Завершение игры → победная сцена

#### 3.2. Система мини-игр
**Сценарии:**
- [ ] Калибровка механика (Roan)
- [ ] Викторина архивариуса (Tel)
- [ ] Испытание старосты (Iar)
- [ ] Выдача наград за успешные мини-игры

#### 3.3. Навигация уровнями
**Сценарии:**
- [ ] Переход от уровня к уровню в цепочке
- [ ] Правильное отображение прогресса
- [ ] Повторное прохождение уровней
- [ ] Активация рычагов завершенных уровней

## Автоматизированные тесты

### Структура тестов
Для GMS2 будем использовать встроенные функции отладки и логирования:

```gml
// Базовый фреймворк тестирования
function test_assert(condition, message) {
    if (!condition) {
        show_debug_message("ТЕСТ НЕ ПРОЙДЕН: " + message);
        return false;
    } else {
        show_debug_message("ТЕСТ ПРОЙДЕН: " + message);
        return true;
    }
}

function test_scenario(name, func) {
    show_debug_message("=== ЗАПУСК СЦЕНАРИЯ: " + name + " ===");
    var result = func();
    if (result) {
        show_debug_message("=== СЦЕНАРИЙ УСПЕШНО ЗАВЕРШЕН: " + name + " ===");
    } else {
        show_debug_message("=== СЦЕНАРИЙ ПРОВАЛЕН: " + name + " ===");
    }
    return result;
}
```

### Тесты для системы прогресса
```gml
function test_progress_system() {
    // Инициализация
    obj_game_manager.init_progress();
    
    // Проверяем начальный прогресс
    test_assert(ds_map_exists(global.game_progress.levels, "1"), "Уровень 1 должен быть включён в начальный прогресс");
    test_assert(global.game_progress.gold == 100, "Стартовое количество золота должно быть 100");
    
    // Тестируем завершение уровня
    obj_game_manager.complete_level(1);
    var level1_data = ds_map_find_value(global.game_progress.levels, "1");
    test_assert(level1_data.completed == true, "Уровень 1 должен быть отмечен как завершённый");
    
    // Тестируем активацию рычага
    obj_game_manager.set_lever_pulled(1, true);
    level1_data = ds_map_find_value(global.game_progress.levels, "1");
    test_assert(level1_data.lever_pulled == true, "Рычаг уровня 1 должен быть активирован");
    
    return true;
}
```

### Тесты для головоломок
```gml
// Тест генерации лабиринта
function test_maze_generation() {
    var maze_obj = instance_create_layer(0, 0, "Instances", obj_maze_puzzle);
    maze_obj.initialize(15, 15); // 15x15 лабиринт
    
    test_assert(array_length_2d(maze_obj.grid, 0) == 15, "Ширина лабиринта должна быть 15");
    test_assert(array_length_2d(maze_obj.grid, 1) == 15, "Высота лабиринта должна быть 15");
    
    // Уничтожаем объект после теста
    instance_destroy(maze_obj);
    return true;
}
```

## Ручное тестирование

### Проверочный список (Checklist)
Для каждого билда игры:

#### Основные функции:
- [ ] Игра запускается без ошибок
- [ ] Главное меню отображается корректно
- [ ] Навигация по меню работает
- [ ] Система сохранений работает (F5 в игре)
- [ ] Загрузка сохранённой игры работает

#### Хаб-город:
- [ ] Игрок может двигаться по городу
- [ ] Взаимодействие с NPC работает (E для общения)
- [ ] Диалоги отображаются корректно
- [ ] Навигация к уровням работает
- [ ] Мини-игры у NPC запускаются

#### Уровни:
- [ ] Все 6 уровней доступны
- [ ] Каждая головоломка функционирует
- [ ] Активация рычагов работает после прохождения
- [ ] Переход между уровнями работает

#### Система прогресса:
- [ ] Прогресс сохраняется между сеансами
- [ ] Открытые уровни остаются открытыми
- [ ] Награды за мини-игры начисляются корректно

## Регрессионное тестирование

После каждого значительного изменения необходимо проверять:

1. **Основной игровой маршрут:** Начало игры → прохождение всех уровней → победа
2. **Система сохранений:** Сохранение, выход, загрузка
3. **Мини-игры NPC:** Проверка всех трех мини-игр
4. **Навигация:** Переходы между всеми состояниями игры
