# Руководство по использованию улучшенной аудио-системы

**Дата обновления**: 16.04.2026  
**Версия**: 2.0

## Обзор изменений

Аудио-система была расширена для поддержки:
- Индивидуальных музыкальных тем для всех 12 уровней
- Контекстных модификаторов громкости
- Плавных переходов между музыкальными темами (кроссфейд)
- Динамической регулировки громкости

## Новые функции

### 1. Контекстные модификаторы громкости

#### apply_audio_context(context_name)
Применяет контекстный модификатор громкости для создания нужной атмосферы.

**Доступные контексты:**
- `"neutral"` - нормальная громкость (music: 1.0, sfx: 1.0)
- `"menu"` - меню (music: 0.8, sfx: 1.0)
- `"town"` - город-хаб (music: 0.7, sfx: 0.9)
- `"puzzle_active"` - активная головоломка (music: 0.5, sfx: 1.0)
- `"puzzle_complete"` - завершение головоломки (music: 0.9, sfx: 1.2)
- `"dialogue"` - диалог с NPC (music: 0.4, sfx: 0.8)
- `"intense_moment"` - напряженный момент (music: 0.9, sfx: 1.1)
- `"victory"` - победа (music: 1.0, sfx: 1.2)

**Пример использования:**
```gml
// При начале головоломки
apply_audio_context("puzzle_active");

// При диалоге с NPC
apply_audio_context("dialogue");

// После завершения головоломки
apply_audio_context("puzzle_complete");
```

### 2. Кроссфейд между музыкальными темами

#### crossfade_music(new_music, fade_duration_seconds)
Плавно переключает музыку с текущей на новую.

**Параметры:**
- `new_music` - индекс нового звукового ресурса
- `fade_duration_seconds` - длительность перехода в секундах (рекомендуется 1-3 секунды)

**Пример использования:**
```gml
// Плавный переход к музыке нового уровня
var new_music = get_sound_resource("snd_cave_new_7");
crossfade_music(new_music, 2.0); // 2 секунды на переход
```

### 3. Плавное изменение громкости

#### lerp_volume(volume_type, target_value, duration_seconds)
Плавно изменяет громкость до целевого значения.

**Параметры:**
- `volume_type` - тип громкости: `"master"`, `"music"`, `"sfx"`
- `target_value` - целевое значение (0.0 - 1.0)
- `duration_seconds` - длительность изменения в секундах

**Пример использования:**
```gml
// Плавно уменьшить музыку перед важным моментом
lerp_volume("music", 0.3, 1.5);

// Вернуть нормальную громкость
lerp_volume("music", 0.7, 1.0);
```

### 4. Получение текущего контекста

#### get_current_audio_context()
Возвращает название текущего аудио-контекста.

**Пример использования:**
```gml
var current_context = get_current_audio_context();
if (current_context == "dialogue") {
    // Диалог активен, не запускать другие звуки
}
```

## Музыкальные темы для уровней

### Ранние пещеры (1-3)
- **Уровень 1 (Лабиринт)**: `snd_cave_early_1` - ambient, эхо, минимализм
- **Уровень 2 (Поиск слов)**: `snd_cave_early_2` - спокойная, размышляющая
- **Уровень 3 (Память)**: `snd_cave_early_3` - ритмичная, паттерны

### Средние пещеры (4-6)
- **Уровень 4 (Пары)**: `snd_cave_mid_4` - игривая, концентрация
- **Уровень 5 (Платформер)**: `snd_cave_mid_5` - динамичная, энергичная
- **Уровень 6 (Финальное испытание)**: `snd_cave_mid_6` - напряженная, эпичная

### Новые пещеры (7-9)
- **Уровень 7 (Загадки Сфинкса)**: `snd_cave_new_7` - древняя, мудрость
- **Уровень 8 (Звуковые Ловушки)**: `snd_cave_new_8` - музыкальная, гармоничная
- **Уровень 9 (Прыгающий Путь)**: `snd_cave_new_9` - быстрая, прыжковая

### Поздние пещеры (10-12)
- **Уровень 10 (Запоминалка)**: `snd_cave_late_10` - медитативная, фокус
- **Уровень 11 (Песнь Пещер)**: `snd_cave_late_11` - хоральная, резонанс
- **Уровень 12 (Финальный Подвиг)**: `snd_cave_finale_12` - героическая, триумф

## Примеры использования

### Пример 1: Переход между уровнями с кроссфейдом
```gml
// В obj_game_manager при смене комнаты
if (room == rm_cave_7) {
    var new_music = get_sound_resource("snd_cave_new_7");
    crossfade_music(new_music, 2.0);
    apply_audio_context("neutral");
}
```

### Пример 2: Управление громкостью во время головоломки
```gml
// В obj_puzzle при начале головоломки
function start_puzzle() {
    apply_audio_context("puzzle_active");
    // Музыка автоматически станет тише
}

// При завершении головоломки
function complete_puzzle() {
    apply_audio_context("puzzle_complete");
    play_event_sound("puzzle_solve");
    // Музыка станет громче, SFX усилятся
}
```

### Пример 3: Диалог с NPC
```gml
// В obj_npc при начале диалога
function start_dialogue() {
    apply_audio_context("dialogue");
    // Музыка станет значительно тише
}

// При завершении диалога
function end_dialogue() {
    apply_audio_context("town");
    // Музыка вернется к нормальной громкости для города
}
```

### Пример 4: Напряженный момент в игре
```gml
// При появлении сложного препятствия
function trigger_intense_moment() {
    apply_audio_context("intense_moment");
    lerp_volume("music", 0.9, 0.5); // Быстрое усиление музыки
}

// После прохождения препятствия
function end_intense_moment() {
    apply_audio_context("neutral");
}
```

## Рекомендации по использованию

### Длительность переходов
- **Быстрые переходы** (0.3-0.5 сек): для UI-звуков, быстрых изменений
- **Средние переходы** (1.0-2.0 сек): для смены музыки между уровнями
- **Медленные переходы** (2.0-4.0 сек): для драматических моментов

### Контексты
- Всегда возвращайте контекст к `"neutral"` после завершения специального события
- Не переключайте контексты слишком часто (минимум 0.5 секунды между переключениями)
- Используйте `get_current_audio_context()` для проверки перед применением нового контекста

### Музыкальные темы
- Используйте `get_current_room_music()` для автоматического определения музыки по комнате
- Для особых моментов используйте `crossfade_music()` вместо `play_music()`
- Проверяйте существование звукового ресурса перед воспроизведением

## Обратная совместимость

Все старые функции продолжают работать:
- `play_music(sound_index)` - прямое воспроизведение музыки
- `play_sound(sound_index)` - воспроизведение звукового эффекта
- `play_event_sound(event_type)` - воспроизведение звука события
- `play_sfx(sound_name)` - совместимость со старым API

## Требования к ресурсам

### Глобальные переменные для новых музыкальных тем
Необходимо определить следующие глобальные переменные в `scr_init_globals.gml`:

```gml
// Ранние пещеры
global.snd_cave_early_1 = -1;
global.snd_cave_early_2 = -1;
global.snd_cave_early_3 = -1;

// Средние пещеры
global.snd_cave_mid_4 = -1;
global.snd_cave_mid_5 = -1;
global.snd_cave_mid_6 = -1;

// Новые пещеры
global.snd_cave_new_7 = -1;
global.snd_cave_new_8 = -1;
global.snd_cave_new_9 = -1;

// Поздние пещеры
global.snd_cave_late_10 = -1;
global.snd_cave_late_11 = -1;
global.snd_cave_finale_12 = -1;
```

## Устранение неполадок

### Музыка не переключается
- Проверьте, что звуковой ресурс существует и загружен
- Убедитесь, что `sound_exists(sound_index)` возвращает `true`
- Проверьте, что глобальная переменная определена и не равна `-1`

### Громкость не изменяется
- Убедитесь, что аудио-система инициализирована (`init_audio_manager()`)
- Проверьте, что `is_audio_available()` возвращает `true`
- Проверьте текущий контекст через `get_current_audio_context()`

### Кроссфейд не работает плавно
- Увеличьте длительность перехода (2-3 секунды)
- Убедитесь, что оба звуковых ресурса существуют
- Проверьте, что не происходит одновременных переключений музыки
