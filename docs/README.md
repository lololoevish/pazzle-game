# Архитектура Adventure Puzzle Game

Этот документ является "источником правды" для архитектуры проекта.

## 📁 Структура проекта

### Логика (Rust/Macroquad) - основная версия
```
src_rust/
├── main.rs           # Точка входа, инициализация macroquad
├── game_state.rs     # GameState, GameProgress, сериализация сохранений
├── scenes/           # Игровые сцены
│   ├── menu.rs       # Главное меню
│   ├── town.rs       # Деревня-хаб и шахтный спуск
│   ├── gameplay.rs   # Пещера уровня: алтарь, рычаг, проход и головоломка
│   ├── victory.rs    # Финальная сцена после шестой печати
│   └── puzzles/      # Головоломки
│       ├── maze.rs
│       ├── wordsearch.rs
│       ├── pattern.rs
│       ├── memory_match.rs
│       ├── platformer.rs
│       └── final_challenge.rs
```

### Ресурсы (Assets)
```
assets/
├── audio/            # Короткие локальные SFX (.wav)
└── sprites/          # Изображения (.png), включая тематический декор поздних пещер
```

## 🛠 Технологический стек (Rust)
- **Движок**: [macroquad](https://macroquad.rs/) (легковесный, быстрый, кроссплатформенный).
- **Сериализация**: `serde` + `serde_json` (для сохранений).
- **Сборка**: Стандартный `cargo` + `.bat` скрипты для Windows.

## Маршрут сцен

Текущий маршрут Rust-версии:

`Menu -> Town -> Playing(текущая_пещера) -> Playing(следующая_пещера) -> ... -> Victory -> Town`

Сцены работают через общий `Scene` trait:

- `handle_input()`
- `update()`
- `draw()`
- `get_next_state()`
- `take_completed_level()`
- `take_progress_update()`

Дополнительно:
- `MenuScene` показывает сводку текущего сохранения и поддерживает подтверждённый сброс «Новая игра».
- `MenuScene` меняет первый пункт между «Играть» и «Продолжить» по наличию сохранения.
- `MenuScene` использует более театральную chapter-подачу: крупный титул, карточку главы и усиленные контрастные панели.
- `MenuScene` использует ужатую вертикальную сетку карточек и внутренние бейджи статуса, чтобы summary и все действия корректно помещались в окно 800x600 без наложений.
- `main.rs` инициализирует локальную SFX-подсистему до запуска игрового цикла.
- `main.rs` теперь ещё и переключает loop-музыку по `GameState`: отдельные темы для меню, деревни, пещер и финала.
- `Cargo.toml` теперь явно включает feature `audio` у `macroquad`, чтобы аудиослой был реально активен в runtime.
- Если аудиобэкенд на машине не поддерживает воспроизведение, `audio.rs` теперь мягко отключает звук без падения игры.
- После полного прохождения `MenuScene` и `TownScene` показывают завершённую экспедицию как отдельное состояние прогресса.
- `TownScene` показывает цель текущей экспедиции, золото, предметы и общий прогресс по печатям прямо в хабе.
- `TownScene` оформлен как драматичный сценический хаб: усиленные световые акценты, отдельная chapter-метка, фокус-панель и более жёсткое разделение слоёв UI.
- `TownScene` делает реплики NPC реактивными не только к общей прогрессии, но и к уже полученным побочным наградам, повторным прохождениям и финальному состоянию экспедиции.
- `GameplayScene` использует level-specific chapter-лейблы, акцентные цвета и усиленные панели, чтобы поздние пещеры и puzzle-shell визуально продолжали стиль меню и хаба.
- Поздние пещеры 3-6 получили собственные декоративные мотивы поверх общей оболочки: часовые кольца, зеркальные панели, кристаллические разломы и пульсирующее ядро.
- Для поздних пещер 3-6 подключены и отдельные локальные PNG-ассеты декора: `clockwork_emblem.png`, `mirror_shard.png`, `crystal_cluster.png`, `core_spire.png`, которые рендерятся через `visual_assets.rs`.
- Меню, хаб, пещеры и финальная сцена используют базовые SFX для навигации, подтверждения, отмены, успеха и рычага.
- Внутренние события пазлов тоже получили SFX: клики и ошибки в pattern/memory, сбор кристаллов и падение в platformer, сбор артефактов, столкновения и тайм-аут в финальном испытании.
- В `assets/audio/` добавлены четыре локальных loop-трека `music_menu.wav`, `music_town.wav`, `music_cave.wav`, `music_victory.wav`; `audio.rs` не только играет SFX, но и централизованно переключает фоновую музыку без перезапуска на каждом кадре.
- У старосты Иара есть отдельное испытание на угадывание числа с собственной наградой.
- В деревне у механика Роана есть отдельная мини-игра-калибровка с собственной наградой.
- У архивариуса Теля есть отдельная мини-викторина по правилам глубин с собственной наградой.
- `VictoryScene` завершает цепочку пещер отдельным эпилогом вместо мгновенного возврата в деревню и использует тот же chapter-style язык для финального summary-экрана.

2. Уровень 2: поиск слов
3. Уровень 3: паттерн-память
4. Уровень 4: поиск пар карточек

## Система сохранений
Игра использует JSON для хранения прогресса.
Файл: `savegame.json`
Структура:
- `levels`: Карта уровней с флагами `completed` и `lever_pulled`.
- `gold`: Игровая валюта.
- `items`: Список предметов.
- `elder_trial_completed`: флаг получения награды за деревенское испытание старосты.
- `mechanic_training_completed`: флаг получения награды за деревенскую мини-игру механика.
- `archivist_quiz_completed`: флаг получения награды за деревенскую викторину архивариуса.

Структура прогресса:
- `completed`: головоломка этой пещеры хотя бы раз решена.
- `lever_pulled`: рычаг внутри этой пещеры опущен, проход к следующему уровню открыт.

Следующая пещера разблокируется только по `lever_pulled`. Если уровень уже был однажды решён, повторный вход в головоломку позволяет нажать `L` и не решать её заново перед рычагом.
Полное прохождение экспедиции определяется по опущенному рычагу шестой пещеры и переводит игру в финальную сцену завершения.
Испытание старосты не влияет на маршрут пещер, но сохраняет отдельную награду за выдержку и точность угадывания.
Мини-игра механика не влияет на разблокировку пещер, но один раз на сохранение выдаёт постоянную награду: золото и предмет.
Викторина архивариуса тоже не влияет на маршрут пещер, но сохраняет отдельную награду за знание правил и структуры экспедиции.

## 🎮 Принципы разработки
1. **Scene Pattern**: переключение между сценами через `GameState` и `Scene`.
2. **Cave Flow**: уровень в Rust-версии это физическая пещера с алтарём, рычагом и проходом в следующую пещеру.
3. **Modular Puzzles**: каждая головоломка - отдельный модуль со своим состоянием.
4. **Rust First**: все новые фичи сначала реализуются в Rust-версии.

## 🔄 План миграции на GameMaker 2
Проект включает подробный план миграции на GameMaker 2, хранящийся в `docs/gamemaker_migration_plan.md`.

### Цель миграции
Создать полноценную версию игры в GameMaker 2 с сохранением всей функциональности Rust-версии:
- 6 уровней с головоломками
- Город-хаб с NPC и мини-играми
- Система прогресса и сохранений
- Аудио- и визуальная составляющая

### Архитектура GameMaker
- **Объекты**: `obj_game_manager`, `obj_player`, `obj_puzzle`, `obj_lever`, `obj_npc`
- **Скрипты**: `scr_game_state`, `scr_save_system`, `scr_audio_manager`, `scr_ui_manager`
- **Комнаты**: `rm_menu`, `rm_town`, `rm_cave_maze`, `rm_cave_archive`, `rm_cave_rhythm`, `rm_cave_pairs`, `rm_cave_platformer`, `rm_cave_final`, `rm_victory`

### Этапы миграции
1. Подготовка и настройка GameMaker 2
2. Импорт ассетов
3. Создание базовых систем
4. Реализация сцен (меню, хаб, уровни)
5. Реализация головоломок
6. Настройка UI/UX
7. Подключение аудио
8. Полировка и оптимизация
9. Сборка и релиз

Подробности смотрите в `docs/gamemaker_migration_plan.md`.
