# 🏗 Архитектура Adventure Puzzle Game

Этот документ является "источником правды" для архитектуры проекта.

## 📁 Структура проекта

### Логика (Rust/Macroquad) — Основная версия
```
src_rust/
├── main.rs           # Точка входа, инициализация macroquad
├── game_state.rs     # Управление игровыми состояниями
├── scenes/           # Игровые сцены
│   ├── menu.rs       # Главное меню
│   ├── town.rs       # Город-хаб (выбор уровней)
│   ├── gameplay.rs   # Общая логика геймплея
│   └── puzzles/      # Головоломки (maze.rs, wordsearch.rs, pattern.rs)
├── entities/         # Игровые объекты (NPC, Игрок)
└── utils/            # Системные утилиты (сохранения, эффекты)
```

### Ресурсы (Assets)
```
assets/
└── sprites/          # Изображения (.png)
```

## 🛠 Технологический стек (Rust)
- **Движок**: [macroquad](https://macroquad.rs/) (легковесный, быстрый, кроссплатформенный).
- **Сериализация**: `serde` + `serde_json` (для сохранений).
- **Сборка**: Стандартный `cargo` + `.bat` скрипты для Windows.

## 💾 Система сохранений
Игра использует JSON для хранения прогресса.
Файл: `savegame.json`
Структура:
- `solved_puzzles`: Список ID пройденных уровней.
- `inventory`: Золото и предметы.
- `stats`: Рекорды и состояние мини-игр.

## 🎮 Принципы разработки
1. **Scene Pattern**: Переключение между сценами через перечисления (Enum).
2. **Modular Puzzles**: Каждая головоломка — это отдельный модуль со своим состоянием.
3. **Rust First**: Все новые фичи сначала реализуются в Rust-версии.
