# Архитектура Adventure Puzzle Game

Этот документ является "источником правды" для архитектуры проекта.

## 📁 Структура проекта

### Логика (Rust/Macroquad) - основная версия
```
src_rust/
├── main.rs           # Точка входа, инициализация macroquad
├── game_state.rs     # GameState, GameProgress, сериализация сохранений
├── scenes/           # Игровые сцены
│   ├── menu.rs       # Главное меню
│   ├── town.rs       # Деревня-хаб и шахтный спуск
│   ├── gameplay.rs   # Пещера уровня: алтарь, рычаг, проход и головоломка
│   ├── victory.rs    # Финальная сцена после шестой печати
│   └── puzzles/      # Головоломки
│       ├── maze.rs
│       ├── wordsearch.rs
│       ├── pattern.rs
│       ├── memory_match.rs
│       ├── platformer.rs
│       └── final_challenge.rs
```

### Ресурсы (Assets)
```
assets/
└── sprites/          # Изображения (.png)
```

## 🛠 Технологический стек (Rust)
- **Движок**: [macroquad](https://macroquad.rs/) (легковесный, быстрый, кроссплатформенный).
- **Сериализация**: `serde` + `serde_json` (для сохранений).
- **Сборка**: Стандартный `cargo` + `.bat` скрипты для Windows.

## Маршрут сцен

Текущий маршрут Rust-версии:

`Menu -> Town -> Playing(текущая_пещера) -> Playing(следующая_пещера) -> ... -> Victory -> Town`

Сцены работают через общий `Scene` trait:

- `handle_input()`
- `update()`
- `draw()`
- `get_next_state()`
- `take_completed_level()`
- `take_progress_update()`

Дополнительно:
- `MenuScene` показывает сводку текущего сохранения и поддерживает подтверждённый сброс «Новая игра».
- `MenuScene` меняет первый пункт между «Играть» и «Продолжить» по наличию сохранения.
- После полного прохождения `MenuScene` и `TownScene` показывают завершённую экспедицию как отдельное состояние прогресса.
- `TownScene` показывает цель текущей экспедиции, золото, предметы и общий прогресс по печатям прямо в хабе.
- В деревне у механика Роана есть отдельная мини-игра-калибровка с собственной наградой.
- `VictoryScene` завершает цепочку пещер отдельным эпилогом вместо мгновенного возврата в деревню.

## Уровни Rust-версии

1. Уровень 1: лабиринт со скольжением до стены
2. Уровень 2: поиск слов
3. Уровень 3: паттерн-память
4. Уровень 4: поиск пар карточек
5. Уровень 5: платформер со сбором кристаллов
6. Уровень 6: финальное испытание на время

## Система сохранений
Игра использует JSON для хранения прогресса.
Файл: `savegame.json`
Структура:
- `levels`: Карта уровней с флагами `completed` и `lever_pulled`.
- `gold`: Игровая валюта.
- `items`: Список предметов.
- `mechanic_training_completed`: флаг получения награды за деревенскую мини-игру механика.

Структура прогресса:
- `completed`: головоломка этой пещеры хотя бы раз решена.
- `lever_pulled`: рычаг внутри этой пещеры опущен, проход к следующему уровню открыт.

Следующая пещера разблокируется только по `lever_pulled`. Если уровень уже был однажды решён, повторный вход в головоломку позволяет нажать `L` и не решать её заново перед рычагом.
Полное прохождение экспедиции определяется по опущенному рычагу шестой пещеры и переводит игру в финальную сцену завершения.
Мини-игра механика не влияет на разблокировку пещер, но один раз на сохранение выдаёт постоянную награду: золото и предмет.

## 🎮 Принципы разработки
1. **Scene Pattern**: переключение между сценами через `GameState` и `Scene`.
2. **Cave Flow**: уровень в Rust-версии это физическая пещера с алтарём, рычагом и проходом в следующую пещеру.
3. **Modular Puzzles**: каждая головоломка - отдельный модуль со своим состоянием.
4. **Rust First**: все новые фичи сначала реализуются в Rust-версии.
