# НОВАЯ МЕХАНИКА: Последовательные переходы между уровнями

## Обзор
Добавлена новая система последовательных переходов, где после прохождения каждого уровня игрок попадает на короткий платформенный переход к следующему уровню, создавая непрерывный игровой процесс.

## Основные изменения

### 1. Последовательная цепочка уровней
- После завершения уровня игрок не возвращается в город, а попадает на платформенный переход к следующему уровню
- Цепочка: Уровень 1 → Переход к уровню 2 → Уровень 2 → Переход к уровню 3 → ... → Уровень 12

### 2. Система переходов
- Создана система `scr_level_transition_platformer.gml` с конфигурацией переходов между уровнями
- Каждый переход содержит параметры: длина, препятствия, собираемые предметы, враги
- Проверка разблокировки следующего уровня перед переходом

### 3. Интеграция с игрой
- Объект игрока `obj_player.gml` теперь имеет метод `complete_level_with_transition()`
- Менеджер игры `obj_game_manager.gml` вызывает новую логику при завершении головоломки
- Решение головоломки теперь ведет к переходу к следующему уровню, а не возврату в город

### 4. Логика завершения
- После прохождения 12-го уровня игрок отправляется в финальную сцену или город
- Сохранение прогресса продолжает работать как обычно
- Система прогресса обновлена до поддержки 12 уровней

## Технические изменения

### Новые файлы:
- `scr_level_transition_platformer.gml` - основная логика переходов между уровнями

### Обновленные файлы:
- `obj_player.gml` - добавлен метод для завершения уровня с переходом
- `obj_game_manager.gml` - обновлена логика завершения головоломки
- `scr_game_state.gml` - возможно, будет обновлена для поддержки цепочки уровней

## Игровой процесс
1. Игрок проходит текущий уровень
2. После завершения головоломки и опускания рычага:
   - Если это не 12-й уровень, игрок перемещается к следующему уровню через платформенный переход
   - Если это 12-й уровень, запускается финальная сцена
3. Это создает непрерывный опыт прохождения всех 12 уровней подряд
