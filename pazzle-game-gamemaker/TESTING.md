# Тестирование проекта Adventure Puzzle Game (GameMaker 2)

## Общая информация

Этот документ описывает сценарии тестирования для проекта Adventure Puzzle Game, реализованного на GameMaker 2. Проект является миграцией Rust-версии игры и должен сохранять ту же функциональность.

## Требования к тестированию

- GameMaker Studio 2 (любая версия 2022+)
- Среда выполнения GameMaker
- Тестовая машина с Windows (основная целевая платформа)

## Сценарии тестирования

### 1. Запуск игры

**Цель:** Проверить успешный запуск игры и отображение главного меню.

**Шаги:**
1. Запустить проект в GameMaker Studio 2
2. Убедиться, что игра запускается без ошибок
3. Проверить, что отображается главное меню
4. Проверить, что воспроизводится музыка главного меню

**Ожидаемый результат:** Игра запускается, отображается главное меню с возможностью выбора опций.

### 2. Система сохранений

**Цель:** Проверить сохранение и загрузку прогресса игры.

**Шаги:**
1. Пройти через главное меню в игру
2. Перейти в несколько пещер
3. Активировать рычаг в одной из пещер
4. Сохранить игру (если реализовано)
5. Выйти и перезапустить игру
6. Загрузить сохранение

**Ожидаемый результат:** Прогресс сохранен и корректно загружается.

### 3. Главное меню

**Цель:** Проверить функциональность главного меню.

**Шаги:**
1. Убедиться, что меню отображается корректно
2. Проверить все кнопки меню:
   - Продолжить (если есть сохранение)
   - Новая игра
   - Сюжет
   - Выход
3. Проверить воспроизведение звуков при навигации
4. Проверить смену музыки при переходе к другим сценам

**Ожидаемый результат:** Все элементы меню работают, звуки воспроизводятся, музыка переключается.

### 4. Город-хаб

**Цель:** Проверить функциональность хаба и взаимодействия с NPC.

**Шаги:**
1. Перейти в город-хаб из меню
2. Проверить перемещение игрока
3. Взаимодействовать с каждым NPC (E):
   - Староста Иара
   - Механик Роан
   - Архивариус Тель
4. Проверить диалоговые окна
5. Проверить отображение прогресса экспедиции

**Ожидаемый результат:** Игрок может перемещаться, взаимодействовать с NPC, отображаются диалоги, прогресс отображается корректно.

### 5. Головоломка "Лабиринт"

**Цель:** Проверить функциональность головоломки лабиринта.

**Шаги:**
1. Перейти в rm_cave_maze
2. Проверить отображение лабиринта
3. Переместить игрока от старта к финишу
4. Проверить скольжение до стены
5. Активировать рычаг при достижении цели
6. Проверить звуковые эффекты

**Ожидаемый результат:** Лабиринт отображается корректно, игрок может пройти к финишу, головоломка решается, звуки воспроизводятся.

### 6. Головоломка "Поиск слов"

**Цель:** Проверить функциональность головоломки поиска слов.

**Шаги:**
1. Перейти в rm_cave_archive
2. Проверить отображение сетки букв
3. Проверить список слов для поиска
4. Выбрать слова на сетке (мышка или клавиши)
5. Проверить, что найденные слова подсвечиваются
6. Найти все слова и активировать рычаг

**Ожидаемый результат:** Сетка отображается, слова можно находить, при нахождении всех слов головоломка считается решенной.

### 7. Головоломка "Ритм"

**Цель:** Проверить функциональность ритм-головоломки.

**Шаги:**
1. Перейти в rm_cave_rhythm
2. Проверить отображение кнопок
3. Наблюдать за цепочкой, которую нужно повторить
4. Повторить цепочку, нажимая клавиши 1-4
5. Пройти несколько раундов
6. Активировать рычаг при успешном прохождении

**Ожидаемый результат:** Головоломка отображается корректно, цепочки генерируются, можно вводить ответы, головоломка решается при правильном прохождении.

### 8. Головоломка "Пары"

**Цель:** Проверить функциональность головоломки поиска пар.

**Шаги:**
1. Перейти в rm_cave_pairs
2. Проверить отображение сетки карточек
3. Открывать карточки, находить пары
4. Проверить, что совпавшие пары остаются открытыми
5. Найти все пары и активировать рычаг

**Ожидаемый результат:** Карточки можно открывать, пары находятся, головоломка решается при нахождении всех пар.

### 9. Головоломка "Платформер"

**Цель:** Проверить функциональность платформер-головоломки.

**Шаги:**
1. Перейти в rm_cave_platformer
2. Проверить управление (WASD или стрелки)
3. Прыгать по платформам
4. Собрать все кристаллы
5. Добраться до цели
6. Активировать рычаг

**Ожидаемый результат:** Игрок может перемещаться и прыгать, можно собрать все кристаллы, головоломка решается при достижении цели с собранными кристаллами.

### 10. Финальная головоломка

**Цель:** Проверить функциональность финальной головоломки.

**Шаги:**
1. Перейти в rm_cave_final
2. Проверить управление
3. Двигаться по уровню, избегая препятствий
4. Собрать все артефакты
5. Добраться до цели
6. Активировать рычаг

**Ожидаемый результат:** Уровень проходим, можно собрать артефакты, головоломка решается при достижении цели с собранными артефактами.

### 11. Финальная сцена

**Цель:** Проверить отображение финальной сцены после завершения экспедиции.

**Шаги:**
1. Завершить все 6 пещер и активировать последний рычаг
2. Убедиться, что происходит переход на финальную сцену
3. Проверить отображение эпилога
4. Проверить возможность возврата в меню или хаб

**Ожидаемый результат:** После завершения шестой пещеры происходит переход на финальную сцену с эпилогом.

## Тестирование аудио-системы

**Цель:** Проверить корректность работы аудио-системы.

**Шаги:**
1. Проверить воспроизведение музыки в каждом сегменте:
   - Меню
   - Хаб
   - Пещеры
   - Финальная сцена
2. Проверить воспроизведение SFX:
   - Подтверждение/отмена
   - Успех/ошибка
   - Взаимодействие с объектами
   - События в головоломках
3. Проверить отсутствие конфликтов между музыкой и SFX
4. Проверить корректное прекращение воспроизведения при смене сцены

**Ожидаемый результат:** Всё аудио воспроизводится корректно, нет наложений, все эффекты слышны.

## Тестирование UI

**Цель:** Проверить корректность работы пользовательского интерфейса.

**Шаги:**
1. Проверить отображение HUD в каждой сцене
2. Проверить отображение прогресса
3. Проверить отображение инвентаря/золота
4. Проверить отображение диалогов с NPC
5. Проверить отображение инструкций в пещерах
6. Проверить отображение сообщений о статусе

**Ожидаемый результат:** Интерфейс отображается корректно, все элементы видны и информативны.

## Сценарии нарушения

**Цель:** Проверить устойчивость игры к нестандартным ситуациям.

**Шаги:**
1. Проверить поведение при резком закрытии игры
2. Проверить поведение при быстрой смене сцен
3. Проверить корректность при неправильном вводе (например, в мини-играх)
4. Проверить поведение при многократном нажатии одних и тех же кнопок
5. Проверить поведение при выходе за границы игровой области (где возможно)

**Ожидаемый результат:** Игра не падает, не возникает критических ошибок, все ситуации обрабатываются корректно.
