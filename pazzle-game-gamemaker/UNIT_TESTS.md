# Модульное тестирование компонентов GameMaker

## Основные компоненты для тестирования

### 1. scr_game_state

**Тестируемые функции:**
- create_new_game_state()
- is_level_completed()
- set_level_completed()
- is_lever_pulled()
- set_lever_pulled()
- is_next_cave_unlocked()
- is_expedition_completed()

**Сценарии тестирования:**

1.1. `create_new_game_state()`
- Проверить, что возвращается структура с правильными полями
- Проверить начальные значения: current_state = "MENU", current_level = 1
- Проверить, что прогресс всех уровней начинается как false
- Проверить, что награды NPC начинаются как false

1.2. `is_level_completed(state, level_num)`
- Проверить, что возвращает false для нового состояния
- Проверить, что возвращает true после вызова set_level_completed
- Проверить корректность проверки разных уровней

1.3. `set_level_completed(state, level_num)`
- Проверить, что правильно устанавливается статус завершения уровня
- Проверить, что не влияет на другие уровни
- Проверить граничные значения (уровни 1-6)

1.4. `is_lever_pulled(state, level_num)`
- Проверить, что возвращает false для нового состояния
- Проверить, что возвращает true после вызова set_lever_pulled

1.5. `set_lever_pulled(state, level_num)`
- Проверить, что правильно устанавливается статус рычага
- Проверить, что не влияет на другие уровни

1.6. `is_next_cave_unlocked(state, level_num)`
- Проверить, что возвращает false если рычаг не опущен
- Проверить, что возвращает true если рычаг опущен
- Проверить, что не может разблокировать уровень 7

1.7. `is_expedition_completed(state)`
- Проверить, что возвращает false для нового состояния
- Проверить, что возвращает true если опущен рычаг 6-го уровня
- Проверить, что возвращает false если опущены рычаги только 1-5 уровней

### 2. scr_save_system

**Тестируемые функции:**
- save_game(game_state)
- load_game()
- reset_game()
- convert_state_for_json(game_state)
- convert_json_to_state(json_data)

**Сценарии тестирования:**

2.1. `save_game(game_state)`
- Проверить, что файл savegame.json создается
- Проверить, что данные корректно записываются в JSON
- Проверить, что структура данных сохраняется полностью

2.2. `load_game()`
- Проверить загрузку из нового файла (должен создать новую игру)
- Проверить загрузку из существующего файла savegame.json
- Проверить, что загруженные данные совпадают с сохраненными

2.3. `reset_game()`
- Проверить, что возвращает новое начальное состояние
- Проверить, что все уровни в начальном состоянии не пройдены

2.4. `convert_state_for_json(game_state)`
- Проверить преобразование ds_map и ds_list в JSON-совместимые структуры
- Проверить, что все поля сохраняются корректно

2.5. `convert_json_to_state(json_data)`
- Проверить преобразование JSON-данных обратно в структуру с ds_map и ds_list
- Проверить, что все поля восстанавливаются корректно

### 3. scr_audio_manager

**Тестируемые функции:**
- initialize_audio()
- play_sfx(sound_name)
- play_music(music_name, looping)
- stop_music()
- set_volume(sfx_vol, music_vol)
- toggle_mute()

**Сценарии тестирования:**

3.1. `initialize_audio()`
- Проверить инициализацию аудио менеджера
- Проверить, что глобальные переменные установлены корректно

3.2. `play_sfx(sound_name)`
- Проверить воспроизведение известных звуков
- Проверить, что не вызывает ошибок при несуществующих звуках
- Проверить приглушенный режим

3.3. `play_music(music_name, looping)`
- Проверить воспроизведение музыки для разных сцен
- Проверить корректное переключение между треками
- Проверить цикличность воспроизведения

3.4. `stop_music()`
- Проверить, что останавливает текущую музыку
- Проверить, что не вызывает ошибок при отсутствии воспроизводимой музыки

3.5. `set_volume(sfx_vol, music_vol)`
- Проверить установку громкости SFX и музыки
- Проверить ограничения значений громкости

3.6. `toggle_mute()`
- Проверить переключение между приглушенным и нормальным режимом
- Проверить, что при включении звука воспроизводится то, что должно

### 4. scr_ui_manager

**Тестируемые функции:**
- show_hud(), hide_hud()
- show_dialog(), hide_dialog()
- show_npc_dialogue(), hide_npc_dialogue()
- show_mini_game(), hide_mini_game()
- draw_ui()
- update_ui()
- complete_mini_game(game_type)

**Сценарии тестирования:**

4.1. `show_hud(), hide_hud()`
- Проверить изменение состояния видимости HUD
- Проверить, что отображается/скрывается соответствующая информация

4.2. `show_dialog(text, choices)`
- Проверить отображение диалогового окна
- Проверить отображение текста и вариантов выбора

4.3. `show_npc_dialogue(npc_name, dialogue_text)`
- Проверить отображение диалога с именем NPC
- Проверить корректное отображение текста

4.4. `show_mini_game(game_type)`
- Проверить установку активного состояния мини-игры
- Проверить правильность типа мини-игры

4.5. `complete_mini_game(game_type)`
- Проверить обновление состояния игры при завершении мини-игры
- Проверить начисление наград за разные типы мини-игр

### 5. Puzzle скрипты

**Тестируемые функции для каждого скрипта:**
- init()
- update()
- draw()
- is_solved()
- solve_puzzle()
- reset()

**Сценарии тестирования:**

5.1. `scr_maze_puzzle`
- Проверить генерацию лабиринта
- Проверить движение игрока
- Проверить скольжение до стены
- Проверить достижение цели
- Проверить решение головоломки

5.2. `scr_word_search_puzzle`
- Проверить генерацию сетки
- Проверить размещение слов
- Проверить механизм выбора слов
- Проверить нахождение всех слов
- Проверить решение головоломки

5.3. `scr_rhythm_puzzle`
- Проверить генерацию последовательности
- Проверить отображение паттерна
- Проверить ввод игроком
- Проверить проверку правильности
- Проверить решение головоломки

5.4. `scr_memory_match_puzzle`
- Проверить генерацию пар
- Проверить отображение карточек
- Проверить механизм выбора
- Проверить сопоставление пар
- Проверить решение головоломки

5.5. `scr_platformer_puzzle`
- Проверить физику игрока
- Проверить столкновения с платформами
- Проверить сбор кристаллов
- Проверить достижение цели
- Проверить решение головоломки

5.6. `scr_final_challenge_puzzle`
- Проверить усложненную физику
- Проверить движущиеся препятствия
- Проверить сбор артефактов
- Проверить сложность, зависящую от прогресса
- Проверить решение головоломки

### 6. scr_puzzle_manager

**Тестируемые функции:**
- init_puzzle_manager()
- register_puzzle_type(type, script_name)
- create_puzzle(type)
- update_puzzle(type)
- draw_puzzle(type, gui_view)
- is_puzzle_solved(type)
- reset_puzzle(type)

**Сценарии тестирования:**

6.1. `init_puzzle_manager()`
- Проверить регистрацию всех типов головоломок
- Проверить создание структур данных

6.2. `create_puzzle(type)`
- Проверить создание головоломки по типу
- Проверить возврат правильной структуры

6.3. `is_puzzle_solved(type)`
- Проверить проверку состояния решения
- Проверить возврат корректных значений

## Рекомендации по тестированию

1. Тестировать каждый компонент изолированно перед интеграцией
2. Использовать mock-объекты для зависимостей при необходимости
3. Проверять граничные условия и ошибочные ситуации
4. Тестировать производительность при частом вызове функций
5. Убедиться, что тесты покрывают как положительные, так и отрицательные сценарии
