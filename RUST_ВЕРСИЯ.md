# 🦀 Rust-версия игры - Полное руководство

## Что это?

Это полная переработка игры на языке программирования **Rust** с использованием игрового движка **macroquad**.

## Зачем Rust?

### Преимущества

1. **⚡ Скорость**: В 10-50 раз быстрее Python
2. **💾 Размер**: Exe файл 5-10 МБ вместо 50+ МБ
3. **🔒 Безопасность**: Защита от утечек памяти и ошибок
4. **📦 Один файл**: Не нужен установленный Python
5. **🌐 Кросс-платформенность**: Windows, Linux, Mac, Web
6. **🎮 Производительность**: Плавная работа даже на слабых ПК

### Недостатки

1. **⏱️ Время сборки**: 1-2 минуты вместо 30 секунд
2. **📚 Сложность**: Rust сложнее Python для изучения
3. **🔧 Настройка**: Требуется установка Rust

## Быстрый старт

### 1. Установка Rust

**Windows:**
```bash
winget install Rustlang.Rustup
```

**Или скачайте**: https://rustup.rs/

### 2. Сборка игры

Запустите `BUILD_RUST.bat` и выберите режим:
- **[1] Разработка** - быстрая сборка для тестов
- **[2] Релиз** - оптимизированная версия для игры

### 3. Запуск

После сборки запустите:
- `target/debug/game.exe` (режим разработки)
- `AdventurePuzzleGame_Rust.exe` (релиз)

```

├── main.rs              # Главный файл, игровой цикл

│   └── mod.rs          # Игрок, предметы, NPC
├── scenes/              # Игровые сцены
│   ├── mod.rs          # Трейт Scene
│   ├── menu.rs         # Главное меню
│   ├── town.rs         # Город-хаб
│   ├── gameplay.rs     # Игровой процесс
│   └── puzzles/        # Головоломки
│       ├── mod.rs
│       ├── maze.rs     # Лабиринт
│       ├── wordsearch.rs # Поиск слов
│       └── pattern.rs  # Память
└── utils/              # Вспомогательные функции
    └── mod.rs
```

## Управление

### Меню и город
- `↑↓` или `WS` - выбор
- `Enter` или `Space` - подтвердить
- `Esc` - назад

### Лабиринт (Уровень 1)
- `WASD` или `↑↓←→` - движение

### Поиск слов (Уровень 2)
- `Клик мышью` - выбор букв

### Память (Уровень 3)
- `Клик мышью` - нажатие кнопок

### Общее
- `F5` - сохранить игру
- `Esc` - выход из уровня

## Технические детали

### Используемые библиотеки

```toml
macroquad = "0.4"      # Игровой движок
serde = "1.0"          # Сериализация
serde_json = "1.0"     # JSON
rand = "0.8"           # Рандом
```

### Оптимизация

В `Cargo.toml` настроено:
```toml
[profile.release]
opt-level = 3          # Максимальная оптимизация
lto = true             # Link Time Optimization
codegen-units = 1      # Лучшая оптимизация
strip = true           # Удаление отладки
```

## Портирование на Web (WASM)

Rust-версию можно запустить в браузере!

```bash
# Установка цели
rustup target add wasm32-unknown-unknown

# Сборка
cargo build --release --target wasm32-unknown-unknown

# Запуск
basic-http-server .
```

Откройте `http://localhost:4000` в браузере!

## Разработка

### Добавление нового уровня

1. Создайте файл в `src_rust/scenes/puzzles/`
2. Реализуйте методы: `new()`, `handle_input()`, `update()`, `draw()`, `is_solved()`
3. Добавьте в `gameplay.rs`

### Добавление NPC

1. Создайте структуру в `src_rust/entities/mod.rs`
2. Добавьте отрисовку в `town.rs`
3. Реализуйте взаимодействие

### Добавление звуков

```rust
// В Cargo.toml добавьте:
macroquad-audio = "0.2"

// В коде:
use macroquad::audio::*;
let sound = load_sound("sound.wav").await.unwrap();
play_sound_once(sound);
```

## FAQ

**Q: Нужно ли удалять Python-версию?**  
A: Нет! Обе версии могут существовать параллельно.

**Q: Какая версия лучше?**  
A: Rust быстрее и компактнее, Python проще для модификации.

**Q: Можно ли играть без сборки?**  
A: Нет, нужно собрать exe через `cargo build --release`.

**Q: Сколько времени занимает сборка?**  
A: Первая сборка: 2-5 минут. Последующие: 10-30 секунд.

**Q: Работает ли на Linux/Mac?**  
A: Да! Просто запустите `cargo build --release`.

**Q: Можно ли запустить в браузере?**  
A: Да! Соберите для WASM (см. раздел выше).
