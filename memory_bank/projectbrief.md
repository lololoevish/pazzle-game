# Adventure Puzzle Game - Project Brief

## 🎯 Цели проекта

Создание 2D приключенческой игры с головоломками и элементами визуальной новеллы, доступной в двух версиях:
- **Python (Pygame)** - для обучения и модификаций
- **Rust (macroquad)** - для продакшена с максимальной производительностью

## 📋 Основные требования

### Функциональные требования

1. **6 уровней с головоломками**:
   - Лабиринт (процедурная генерация)
   - Поиск слов (русские слова)
   - Память/Паттерны (Simon Says)
   - Головоломка на память
   - Платформер с прыжками
   - Финальный уровень с таймером

2. **Город-хаб** - центральная локация для выбора уровней

3. **NPC с мини-играми**:
   - Угадай число
   - Камень-ножницы-бумага
   - Викторина

4. **Система инвентаря** - золото, предметы, награды

5. **Автосохранение** прогресса

6. **Система диалогов** с портретами персонажей

### Технические требования

- **Python версия**: Python 3.7+, Pygame 2.5.2, PyInstaller для сборки
- **Rust версия**: Rust 1.70+, macroquad 0.4, serde для сохранений
- **Размер exe**: Python ~50-80 МБ, Rust ~5-10 МБ
- **FPS**: Стабильные 60 FPS

## 🎮 Игровая механика

### Основной геймплей

- Игрок проходит уровни с головоломками
- За победу получает золото и предметы
- В городе может взаимодействовать с NPC
- Прогресс сохраняется автоматически

### Управление

- **Меню**: ↑↓ или W/S - выбор, Enter - подтверждение
- **В игре**: Стрелки или WASD - движение
- **Взаимодействие**: Мышь
- **Назад/Выход**: ESC
- **Сохранение**: F5

## 🛠 Редактирование в GameMaker 2

Для редактирования игры в GameMaker Studio 2:

### Установка и настройка
1. Установите GameMaker Studio 2 (версия 2022.6 или новее)
2. Откройте проект `gamedata/adventure_puzzle/adventure_puzzle.yyp`

### Структура проекта GameMaker
- **Objects** - игровые объекты (игрок, враги, препятствия)
- **Rooms** - уровни и экраны игры
- **Sprites** - графические ресурсы
- **Sounds** - аудиофайлы
- **Scripts** - игровые функции и логика
- **Shaders** - эффекты рендеринга
- **Fonts** - шрифты
- **Timelines** - анимации по времени
- **Paths** - пути движения

### Основные скрипты
- `game_init()` - инициализация игры
- `level_manager()` - управление уровнями
- `player_control()` - управление игроком
- `save_system()` - система сохранения

### Редактирование уровней
1. Откройте нужный `room_` в папке Rooms
2. Используйте Layer panel для управления слоями:
   - Background - фон
   - Instances - объекты
   - Tiles - тайлы
   - Assets - статические объекты
3. Добавляйте объекты из Objects через Instance Creation

### Создание новых уровней
1. Создайте новый Room через Create > Room
2. Настройте размеры (обычно 800x600)
3. Добавьте необходимые слои
4. Установите background и tileset
5. Добавьте начальные объекты (игрок, выход, врагов и т.д.)

### Работа с графикой
1. Импортируйте спрайты через Create > Sprite
2. Установите размеры и количество фреймов
3. Настраивайте origin и bound box для корректного отображения
4. Привязывайте спрайты к объектам в Object Properties

### Система сохранений
Используется GameMaker Save System:
- `game_save()` - сохранение в save0-9
- `game_load()` - загрузка сохранения
- Сохраняются переменные экземпляров и глобальные переменные

## 🗺 План разработки

### Текущий этап

1. Хаб как игровая прослойка между пещерами уже закрыт по функционалу.
2. Базовая реактивность диалогов по наградам и финалу уже подключена; основной фокус смещен на звук и дальнейшую полировку.
3. Визуальная согласованность уже протянута от меню и деревни до пещер и финала; звук теперь тоже вошёл в активную стадию.
4. Поздние пещеры 3-6 уже получили собственные декоративные мотивы и тематические спрайты, поэтому следующий шаг по визуалу — добивать ранние уровни и общий баланс всей цепочки.
5. Активная миграция на GameMaker 2: созданы архитектурные документы, план переноса логики, документы тестирования и технические задания на перенос функционала.

2. Протянуть тематические ассеты и на ранние уровни.
   Ожидаемый результат: визуальный язык всех шести пещер будет не только цельным, но и более равномерно проработанным по всей экспедиции.
3. Решить, углублять ли диалоги дальше в сторону VN-подачи с портретами.
   Ожидаемый результат: понятная граница между текущей реактивной системой и следующей ступенью сценической подачи.
4. Продолжить реализацию миграции в GameMaker 2 согласно архитектурному плану.
   Ожидаемый результат: реализация основных игровых механик и систем в GMS2.
