# Adventure Puzzle Game - Project Brief

## 🎯 Цели проекта

Создание 2D приключенческой игры с головоломками и элементами визуальной новеллы, доступной в двух версиях:
- **Python (Pygame)** - для обучения и модификаций
- **Rust (macroquad)** - для продакшена с максимальной производительностью

## 📋 Основные требования

### Функциональные требования

1. **6 уровней с головоломками**:
   - Лабиринт (процедурная генерация)
   - Поиск слов (русские слова)
   - Память/Паттерны (Simon Says)
   - Головоломка на память
   - Платформер с прыжками
   - Финальный уровень с таймером

2. **Город-хаб** - центральная локация для выбора уровней

3. **NPC с мини-играми**:
   - Угадай число
   - Камень-ножницы-бумага
   - Викторина

4. **Система инвентаря** - золото, предметы, награды

5. **Автосохранение** прогресса

6. **Система диалогов** с портретами персонажей

### Технические требования

- **Python версия**: Python 3.7+, Pygame 2.5.2, PyInstaller для сборки
- **Rust версия**: Rust 1.70+, macroquad 0.4, serde для сохранений
- **Размер exe**: Python ~50-80 МБ, Rust ~5-10 МБ
- **FPS**: Стабильные 60 FPS

## 🎮 Игровая механика

### Основной геймплей

- Игрок проходит уровни с головоломками
- За победу получает золото и предметы
- В городе может взаимодействовать с NPC
- Прогресс сохраняется автоматически

### Управление

- **Меню**: ↑↓ или W/S - выбор, Enter - подтверждение
- **В игре**: Стрелки или WASD - движение
- **Взаимодействие**: Мышь
- **Назад/Выход**: ESC
- **Сохранение**: F5

## 🗺 План разработки

### Текущий этап

1. Хаб как игровая прослойка между пещерами уже закрыт по функционалу.
2. Базовая реактивность диалогов по наградам и финалу уже подключена; основной фокус смещён на звук и дальнейшую полировку.
3. Визуальная согласованность уже протянута от меню и деревни до пещер и финала; звук теперь тоже вошёл в активную стадию.
4. Поздние пещеры 3-6 уже получили собственные декоративные мотивы и тематические спрайты, поэтому следующий шаг по визуалу — добивать ранние уровни и общий баланс всей цепочки.

### Ближайший план

1. Развить текущий музыкальный слой в более выразительную звуковую драматургию.
   Ожидаемый результат: у уже существующих сценических тем появится более точный баланс, вариативность и лучшая привязка к состояниям игры.
2. Протянуть тематические ассеты и на ранние уровни.
   Ожидаемый результат: визуальный язык всех шести пещер будет не только цельным, но и более равномерно проработанным по всей экспедиции.
3. Решить, углублять ли диалоги дальше в сторону VN-подачи с портретами.
   Ожидаемый результат: понятная граница между текущей реактивной системой и следующей ступенью сценической подачи.

### Следующий этап после этого

1. Свести release-полировку в единый проход.
   Ожидаемый результат: выровненные тексты, HUD, переходы, награды и финальные состояния.
2. Подготовить проект к публичной демонстрации.
   Ожидаемый результат: актуальная документация, понятный статус deliverables и чистый маршрут первого прохождения.
