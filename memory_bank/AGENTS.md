# 🤖 AGENTS.md - Правила агента

Этот файл определяет правила и стандарты для ИИ-агентов, работающих над проектом **Adventure Puzzle Game**.

## 🎯 Общие принципы

1.  **Memory Bank — это святыня**: При каждом новом сеансе или крупном изменении агент должен сверяться с папкой `memory_bank/`.
2.  **Автономность**: Агент должен стремиться к решению задач самостоятельно, используя доступные инструменты (Rust, Python, Bash).
3.  **Двуязычность**: Поддержка как Python версии (для обучения), так и Rust версии (для продакшена).

## 🛠 Технические стандарты

### Rust (Приоритет)
- Использование фреймворка `macroquad`.
- Разделение на модули в `src_rust/`.
- Обработка ошибок через `Result` и `Option`.
- Минификация размера exe (отключение лишних фич).

### Python
- Соблюдение PEP 8.
- Использование `pygame`.
- Обеспечение совместимости для сборки через PyInstaller.

## 🔄 Workflow

1.  **Планирование**: Перед написанием кода агент создает или обновляет план в `plans/`.
2.  **Реализация**: Написание чистого, документированного кода.
3.  **Документирование**: Обновление `activeContext.md` и `progress.md` после каждого значимого этапа.
4.  **Фиксация**: Коммит изменений с четким описанием (conventional commits).

## 🚫 Запреты

- Не удалять файлы без подтверждения (кроме этапа "Очистка проекта").
- Не менять архитектуру без обновления `systemPatterns.md`.
- Не игнорировать предупреждения линтера в Rust.
