# Игровые Сущности

## Обзор

Актуальные игровые сущности находятся в `pazzle-game-gamemaker/objects/`. Исторические Python/Rust сущности не считаются основным runtime-кодом проекта.

## Основные объекты GMS2

### `obj_player.gml`

Назначение: управление игроком, перемещение, взаимодействия, переходы и часть puzzle-flow.

Ключевые особенности:

- режимы движения `topdown` и `platformer`;
- буфер прыжка, coyote time и variable jump height;
- взаимодействие с `obj_interactable`, `obj_npc`, `obj_puzzle` и переходами;
- безопасные helper-вызовы для UI и аудио.

### `obj_game_manager.gml`

Назначение: общий цикл состояния игры и маршрутизация состояний.

Ключевые особенности:

- состояния `menu`, `town`, `playing_level_1..12`, `victory`;
- обработка завершения пазла и рычага;
- запуск переходов и сохранение прогресса.

### `obj_npc.gml`

Назначение: NPC, побочные мини-игры, реактивные диалоги и VN-интеграция.

Ключевые особенности:

- активности старосты, механика и архивариуса;
- награды через общую модель прогресса;
- зависимость реплик от состояния прохождения.

### `obj_puzzle.gml`

Назначение: контейнер активной головоломки и связка с puzzle scripts.

### `obj_lever.gml`

Назначение: рычаг завершения пещеры и открытия следующего шага маршрута.

### `obj_interactable.gml`

Назначение: базовые интерактивные точки мира.

## Прогресс и инвентарь

Прогресс хранится в `global.game_progress`:

```gml
global.game_progress = {
    levels: [
        {completed: false, lever_pulled: false}
    ],
    gold: 100,
    items: [],
    mechanic_training_completed: false,
    archivist_quiz_completed: false,
    elder_trial_completed: false
};
```

Массив `levels` должен соответствовать 12-уровневой модели проекта.
