# Legacy Reference

## Обзор

Этот раздел фиксирует исторические части репозитория, которые больше не являются основной продуктовой целью.

## Исторические ветки

### Python/Pygame

Статус: historical reference.

Python/Pygame-материалы могут сохраняться как ранний обучающий пример, но не определяют текущий продуктовый scope, архитектуру или готовность проекта.

### Rust/macroquad

Статус: historical reference.

Rust-ветка полезна как источник идей по маршруту, UX и прошлым решениям, но основной runtime проекта находится в `pazzle-game-gamemaker/`.

## Текущая основная ветка

Основной продуктовый фокус: GameMaker Studio 2.

Канонические источники:

- `docs/README.md` - верхнеуровневая архитектура;
- `memory_bank/projectbrief.md` - scope, deliverables и процент готовности;
- `memory_bank/activeContext.md` - текущий рабочий фокус;
- `memory_bank/progress.md` - история изменений и известные проблемы.

## Рекомендации

1. Не использовать Python/Rust-документы как источник актуального scope без сверки с `docs/README.md`.
2. Не переносить legacy-формулировки про 6 уровней в новые GMS2-документы.
3. При изменении архитектуры GMS2 сначала обновлять `docs/README.md`, затем Memory Bank.
