# 🔧 Руководство для разработчика

## 🎮 Как добавить новый уровень

### 1. Откройте `src/scenes/gameplay.py`

### 2. Добавьте конфигурацию уровня в метод `load_level()`

```python
elif level == 7:  # Новый уровень
    self.switches = [
        Switch(200, 300, 1, [1, 2, 3]),
        Switch(400, 300, 2, [1, 2, 3]),
        Switch(600, 300, 3, [1, 2, 3])
    ]
    self.obstacles = [
        MovingObstacle(300, 400, 100, 0, 2)
    ]
    self.exit_door = pygame.Rect(700, 500, 50, 80)
```

### 3. Обновите `src/main.py`

Добавьте уровень в `game_progress`:
```python
game_progress = {
    1: {'completed': False, 'lever_pulled': False},
    # ... существующие уровни ...
    7: {'completed': False, 'lever_pulled': False}  # Новый
}
```

### 4. Обновите `src/scenes/town.py`

Добавьте дверь для нового уровня в метод `create_buildings()`:
```python
self.level_doors.append({
    'rect': pygame.Rect(x, y, 60, 80),
    'level': 7,
    'name': 'Уровень 7'
})
```

## 🎲 Как добавить нового NPC с мини-игрой

### В файле `src/scenes/town.py`, метод `__init__()`:

```python
self.npcs = [
    # Существующие NPC...
    NPC(650, 400, "Новый NPC", "Описание", "guess_number", "Награда")
]
```

### Типы мини-игр:
- `"guess_number"` - Угадай число
- `"rock_paper_scissors"` - Камень-ножницы-бумага  
- `"quiz"` - Викторина

### Добавить новую мини-игру:

Откройте `src/scenes/minigames.py` и добавьте новый тип в класс `MiniGameScene`.

## 💾 Система сохранений

### Что сохраняется:
```python
{
    'progress': {
        1: {'completed': True, 'lever_pulled': False},
        # ... для каждого уровня
    },
    'inventory': {
        'gold': 150,
        'items': ['Золотой ключ', 'Амулет'],
        'won_minigames': ['Торговец', 'Мудрец']
    }
}
```

### Где хранится:
- При запуске из исходников: `./savegame.json`
- При запуске из exe: `savegame.json` рядом с exe

### Ручное управление сохранениями:

```python
from utils.save_system import save_game, load_game, delete_save, has_save

# Проверить наличие сохранения
if has_save():
    print("Сохранение найдено!")

# Загрузить
data = load_game()
if data:
    progress, inventory = data

# Сохранить
save_game(game_progress, player_inventory)

# Удалить
delete_save()
```

## 🏗️ Архитектура проекта

### Игровой цикл (src/main.py):
```
Инициализация → Игровой цикл → Сохранение → Выход
                     ↓
        ┌────────────┴────────────┐
        ↓                         ↓
    Обработка событий      Обновление состояния
        ↓                         ↓
    Отрисовка              Ограничение FPS
```

### Состояния игры:
- `STATE_MENU` - Главное меню
- `STATE_TOWN` - Город (хаб)
- `STATE_PLAYING` - Игровой процесс
- `STATE_MINIGAME` - Мини-игра

### Переходы между состояниями:
```
MENU → TOWN → PLAYING → TOWN
         ↓
    MINIGAME → TOWN
```

## 🐛 Отладка

### Логирование:
Все логи записываются в `game.log`:
```python
import logging
logger = logging.getLogger()
logger.info("Сообщение для отладки")
logger.error("Ошибка!")
```

### Консольный режим:
В `AdventurePuzzleGame.spec` измените:
```python
console=False,  # на
console=True,   # для отладки
```

### Запуск без сборки:
```bash
python src/main.py
```
Быстрее для тестирования изменений!

## 📦 Оптимизация размера exe

### Исключить ненужные модули:
В `AdventurePuzzleGame.spec`:
```python
excludes=['tkinter', 'matplotlib', 'numpy'],
```

### Отключить UPX (если проблемы):
```python
upx=False,
```

### Создать папочную сборку (меньше размер):
Замените `EXE` на:
```python
coll = COLLECT(
    exe,
    a.binaries,
    a.zipfiles,
    a.datas,
    strip=False,
    upx=True,
    name='AdventurePuzzleGame'
)
```

## 🚀 Советы по производительности

1. **Кэшируйте поверхности**: Не создавайте Surface каждый кадр
2. **Используйте dirty rect**: Обновляйте только изменённые области
3. **Ограничьте частицы**: Не создавайте слишком много объектов
4. **Оптимизируйте коллизии**: Используйте пространственное разбиение
